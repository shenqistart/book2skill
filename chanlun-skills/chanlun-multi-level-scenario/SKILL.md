# 缠论多级别情景推演

## R — 来源

依据《教你炒股票20：走势中枢级别扩张及第三类买卖点》和《教你炒股票29：转折的力度与级别》。

## I — 方法论骨架

缠论最容易被误用的地方，是把小级别信号直接放大成大级别结论。
某级别背驰后，不只有“反转”一种结果。
常见情景包括：只触及最后中枢附近的弱反弹、形成更大级别中枢、进入同级别或更高级别反趋势。
多级别推演的目标是提前列出条件，而不是提前押注答案。
一个合格输出必须同时写出成立条件、失效条件和下一步观察点。

## A1 — 书中的应用

### 案例: 5分钟下跌背驰后的三类结果

- **问题**: 5分钟级别下跌背驰后，反弹到底意味着什么。
- **方法论的使用**: 作者把后续分为最后中枢扩展、更大级别盘整、反趋势等情景。
- **结论**: 背驰后要看反弹是否回到最后中枢以及后续结构。
- **结果**: 避免一看到背驰就断言大反转。

## A2 — 触发场景

1. 用户说大级别和小级别信号冲突。
2. 用户问背驰后是反弹、盘整还是反转。
3. 用户要求做“如果 A 则 B，如果失败则 C”的预案。

### 语言信号

- “日线还没走完，但30分钟背驰”
- “这只是反弹还是反转”
- “多级别怎么看”
- “给我三种情景预案”

### 与相邻 skill 的区分

- 与 `chanlun-divergence-diagnosis` 的区别: 背驰诊断只回答是否背驰；本 skill 处理背驰后的多级别演化。
- 与 `chanlun-buy-sell-points` 的区别: 本 skill 输出情景树，不判断单个买卖点是否成立。

## E — 可执行步骤

1. **列出所有相关级别**
   - 完成标准: 至少区分大级别、操作级别、次级别。

2. **锁定上一级中枢和最后中枢**
   - 完成标准: 明确观察“是否回到/离开/跌回中枢”的关键位置。

3. **生成三种情景**
   - 完成标准:
     - 弱反弹/弱回落。
     - 更大级别中枢或盘整。
     - 反趋势。

4. **为每种情景列确认信号**
   - 完成标准: 每个情景至少有 2 个结构确认条件。

5. **给出下一步数据需求**
   - 完成标准: 明确下一根/下一段/下一中枢要观察什么。

## B — 边界

- 不承诺哪种情景一定发生。
- 不把小级别背驰直接当成大级别反转。
- 不在没有级别信息时做多级别推演。

### 失败模式

- 用单一结论替代情景树。
- 忽略最后一个中枢的位置。
- 只写看多/看空，不写失效条件。

### 盲点 / 时代局限

- 多级别推演容易变复杂，必须控制输出为可操作的三种情景。
- 对真实交易还需叠加仓位、止损、市场流动性和交易成本。
