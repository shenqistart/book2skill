## 1. 结构

### 类型

证券市场技术分析方法论、走势结构分解框架、交易纪律训练体系。

### 一句话主旨

《缠论》试图把价格走势拆成可递归识别的结构，并用“级别、中枢、背驰、三类买卖点”建立一套当下分析和交易计划框架。

### 骨架

1. 走势不是随意波动，而是可以按级别递归分解。
2. 最小识别单位从 K 线处理开始，逐步形成分型、笔、线段。
3. 中枢是价格重叠区间，是判断盘整、趋势、延续、扩张的核心。
4. 买卖点不是主观预测，而是与中枢、级别、背驰关系绑定的结构位置。
5. 背驰用于判断趋势力度衰竭，但必须放回级别和中枢结构里确认。
6. 任何分析必须区分“当下可确认”与“事后看清楚”，否则容易把理论变成玄学。

**论点关系**: 递进。先定义几何结构，再定义中枢，再推出买卖点与转折。

### 作者要解决的核心问题

在不依赖消息、情绪和主观猜测的情况下，如何用价格结构识别走势所处阶段，并制定可执行、可复盘的买卖计划。

## 2. 解释

### 关键术语

| 术语 | 在本 skill 中的工作定义 | 与常识用法的差异 |
|---|---|---|
| 级别 | 用来观察走势的递归层级，如 1 分钟、5 分钟、30 分钟、日线等 | 不是“时间越大越重要”的简单排序，而是分析递归关系 |
| 分型 | 由处理过包含关系后的 K 线形成的顶/底识别单元 | 不是随便看见高低点就算 |
| 笔 | 顶分型与底分型之间的基本连接 | 不是任意两点连线 |
| 线段 | 至少由三笔构成，并满足重叠/破坏规则的结构 | 不是视觉上的一段趋势线 |
| 中枢 | 至少三个连续次级别走势类型的重叠区间 | 不是普通支撑阻力位 |
| 背驰 | 后一段同向走势力度弱于前一段，同级别中枢关系要成立 | 不是指标背离的简单同义词 |
| 第一类买卖点 | 趋势背驰后形成的转折点 | 不是“抄底摸顶”的感觉 |
| 第二类买卖点 | 第一类买卖点后的次级别回调/反抽确认点 | 不是任何回踩都算 |
| 第三类买卖点 | 离开中枢后回试不回到中枢的延续点 | 不是突破后随便追 |

### 核心命题

1. 走势分析必须先定级别，否则所有判断都会混乱。
2. 中枢是缠论的地图中心，买卖点必须围绕中枢关系定义。
3. 背驰只说明某级别走势力量衰竭，后续转折力度还要继续分类。
4. 第三类买卖点适合判断趋势延续，但也可能进入更大级别盘整。
5. 真正的操作价值来自“结构确认 + 预案 + 止损/退出条件”，不是单点预测。

### 论证链

公开章节中，作者从“走势终完美”出发，先提出走势类型与中枢，再用中枢定义盘整/趋势；随后讨论三类买卖点的完备性、背驰与 MACD 辅助判断、背驰后的转折力度，以及分型、笔、线段的基础划分规则。

## 3. 批判

### 时代局限

- 原始文本主要来自 A 股早期市场语境，交易制度、信息速度、算法交易和流动性结构已经变化。
- 很多案例依赖当时个股和指数环境，不能直接迁移成今天的收益承诺。
- 原文表达带有强烈个人风格，部分断言需要在实盘统计中重新验证。

### 立场盲点

- 对操作者执行力要求极高，普通用户容易把“唯一划分”变成事后重画。
- 对交易成本、滑点、停牌、涨跌停、流动性冲击的系统化处理不足。
- 对风险预算、仓位管理、组合暴露的讨论不如结构分析细。

### 未被证明的假设

- 结构识别的一致性可以由训练达到足够稳定。
- 背驰后的可交易空间足以覆盖成本和风险。
- 多级别递归分析不会因主观选择观察级别而产生确认偏误。

### 最强反对意见

缠论可能给了交易者一套高度复杂的解释语言，但若没有严格标注规则、前视验证和统计检验，使用者很容易在行情走完后重构“正确划分”，从而误以为系统具有稳定预测力。
