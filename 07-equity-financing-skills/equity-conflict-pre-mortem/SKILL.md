# 股权冲突预演

## R — 原文 (Reading)

> “控制权争夺”
>
> 来源: 苏龙飞《股权战争》序言。

## I — 方法论骨架 (Interpretation)

这个 skill 把股权设计从“现在能签吗”推进到“未来哪一刻会爆雷”。《股权战争》以控制权争夺为核心, 复盘创始人、投资人、董事会、夫妻/亲属、政府环境和资本条款之间的冲突。它提醒创业者, 很多股权战争不是突然发生, 而是早期协议、认知盲区和权责不匹配留下的种子。

预演时不只问谁对谁错, 而要问: 当公司变好、变坏、融资、上市、并购、现金紧张、创始人离开或夫妻关系变化时, 哪些条款会被激活, 哪些人有阻断权, 哪些人有动机改变立场。

## A1 — 书中的应用 (Past Application)

### 案例 1: 雷士照明式控制权冲突

- **问题**: 创始人与资本方在公司发展和控制权上发生激烈冲突。
- **方法论的使用**: 复盘董事会、投资条款、创始人角色和资本方权力。
- **结论**: 控股权争夺常由治理结构和资本规则认知不足放大。
- **结果**: 融资前就要预演资本进入后的权力变化。

### 案例 2: 支付宝/阿里相关争议

- **问题**: 监管、牌照、控制权和股东利益交织。
- **方法论的使用**: 把商业安排放入监管和股东关系中分析。
- **结论**: 外部制度变化可能触发股东冲突。
- **结果**: 重大监管依赖业务必须提前写清决策和补偿机制。

### 案例 3: 夫妻店与亲属股权

- **问题**: 亲密关系进入股权结构, 冲突时公司治理失灵。
- **方法论的使用**: 预演婚姻、继承、离职、竞业和退出场景。
- **结论**: 情感信任不能替代权责和退出规则。
- **结果**: 亲属创业也必须写清边界。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户准备签合伙人协议、股东协议、婚内共同创业安排或融资协议。
2. 用户担心创始人、投资人、董事会或亲属未来翻脸。
3. 用户已经出现股东矛盾, 想找冲突源头。
4. 用户需要在重大融资、并购、上市或控制权变化前做风险预演。

### 语言信号

- “以后会不会抢控制权?”
- “股东冲突怎么提前避免?”
- “夫妻一起创业股权怎么防风险?”
- “投资人条款会不会埋雷?”

### 与相邻 skill 的区分

- 与 `founder-control-threshold-map` 的区别: 前者画当前权力地图; 本 skill 在未来事件中预演冲突。
- 与 `term-sheet-economics-control-audit` 的区别: 前者审 TS 条款; 本 skill 把条款放进未来冲突场景。

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **列利益相关者**
   - 完成标准: 至少列出创始人、投资人、员工股东、董事、高管、亲属、债权人和监管相关方。

2. **选择压力场景**
   - 完成标准: 选择增长过快、业绩下滑、再融资、并购退出、创始人离开、婚姻/继承变化、监管变化等场景。

3. **映射权力触发器**
   - 完成标准: 对每个场景标出会触发的条款、表决权、董事会权力、回购/退出和违约责任。

4. **识别冲突种子**
   - 完成标准: 标出权责不匹配、收益不匹配、信息不对称、退出不清、亲属关系混入治理五类风险。

5. **输出修补清单**
   - 完成标准: 给出应补写、应谈判、应披露、应律师复核和应暂缓签署的事项。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要求具体诉讼策略、取证方案或规避法律责任。
- 用户只需要基础估值和稀释计算。
- 用户试图用控制权设计侵害小股东、配偶或员工的合法权益。

### 作者在书中警告的失败模式

- 创业者不懂资本游戏规则就签协议。
- 只靠关系和信任, 不写退出和冲突解决机制。
- 成熟公司治理规则与创始人个人控制欲发生冲突。

### 作者的盲点 / 时代局限

- 书中案例受具体时代、司法环境和监管背景影响。
- 案例不能直接推出当前法律结论, 只能作为风险预演素材。

### 容易混淆的邻近方法论

- “危机公关”: 本 skill 聚焦股权和治理结构, 不处理舆情。
- “律师审查”: 本 skill 输出问题清单, 不替代法律意见。
