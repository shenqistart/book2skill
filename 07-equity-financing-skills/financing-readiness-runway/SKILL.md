# 融资准备与 Runway 判断

## R — 原文 (Reading)

> “演示的数据”
>
> 来源: 史蒂文·霍夫曼《穿越寒冬》第一章。

## I — 方法论骨架 (Interpretation)

这个 skill 把“我要融资”拆成三个问题: 是否需要融资, 是否准备好融资, 以及现金是否撑得到融资完成。

融资不是在缺钱那天才启动。创业者需要先证明客户、收入、增长、留存、单位经济或关键里程碑, 再决定用融资放大什么。书中强调, 未来给投资人看的数据, 往往就是现在应当开始收集和改善的数据。

它还要求反问: 是否可以通过客户预付款、收入、成本压缩、债务或阶段性目标推迟融资; 如果必须融资, 融资金额应覆盖到下一次可验证的价值跃迁, 而不是只覆盖眼前账单。

## A1 — 书中的应用 (Past Application)

### 案例 1: 在融资前先准备客户证据

- **问题**: 创业者只准备故事, 但缺少用户和收入数据。
- **方法论的使用**: 把路演材料反推为日常经营指标。
- **结论**: 现在收集的数据, 决定未来融资叙事是否可信。
- **结果**: 融资准备变成持续经营动作, 而不是临时包装。

### 案例 2: Runway 和融资周期

- **问题**: 公司现金只够两三个月, 才开始寻找投资人。
- **方法论的使用**: 把融资周期、尽调、谈判和到账时间纳入现金计划。
- **结论**: 现金不足会削弱谈判力, 甚至迫使接受坏条款。
- **结果**: 提前规划融资窗口和降本方案。

### 案例 3: 不一定先拿股权融资

- **问题**: 创业者把融资等同于卖股权。
- **方法论的使用**: 检查客户承诺、收入、合作、债务和延后支出的可能性。
- **结论**: 股权融资是高成本资金, 不应成为唯一默认选项。
- **结果**: 在融资前先探索非稀释性资金和业务证据。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户说公司快没钱了, 问现在能不能融资。
2. 用户准备路演材料, 但不知道该展示哪些指标。
3. 用户想知道融多少钱合适。
4. 用户把融资当成挽救业务的唯一办法。

### 语言信号

- “现在该不该融资?”
- “现金只够几个月怎么办?”
- “BP 要准备哪些关键数据?”
- “融资金额怎么定?”

### 与相邻 skill 的区分

- 与 `investor-fit-signal-filter` 的区别: 本 skill 判断自身准备度; 后者筛投资人匹配度。
- 与 `valuation-dilution-clarifier` 的区别: 本 skill 先判断是否进入融资; 后者处理估值和稀释语言。

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **计算 runway**
   - 完成标准: 估算当前现金、月 burn、保守收入、必要支出和最坏情况下剩余月份。

2. **定义下一里程碑**
   - 完成标准: 写明融资要支撑到哪个可验证节点, 如收入、留存、产品发布、监管许可或关键客户。

3. **检查融资证据**
   - 完成标准: 至少列出市场、客户、产品、增长、财务、团队和风险七类证据的完整度。

4. **比较资金路径**
   - 完成标准: 比较股权融资、客户预付款、收入自循环、债务、裁撤成本和战略合作的利弊。

5. **输出融资窗口**
   - 完成标准: 给出“现在启动 / 补证据后启动 / 暂缓 / 紧急生存模式”, 并说明最脆弱假设。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户已经拿到 Term Sheet, 需要条款审计。
- 用户要求承诺融资成功、包装虚假数据或隐瞒风险。
- 用户需要实时市场估值倍数或投资机构最新偏好。

### 作者在书中警告的失败模式

- 太晚开始融资, 导致现金压力压低谈判力。
- 只讲愿景, 没有客户和业务数据。
- 把不适合的资金当作救命钱。

### 作者的盲点 / 时代局限

- 融资环境、利率和投资人偏好高度周期性, 需要用当前市场信息复核。
- 不同行业的里程碑差异很大, 不能机械套用互联网创业模板。

### 容易混淆的邻近方法论

- “BP 美化”: 本 skill 反对包装, 强调真实证据。
- “财务预测”: 本 skill 使用预测, 但重点是融资准备度和现金窗口。
