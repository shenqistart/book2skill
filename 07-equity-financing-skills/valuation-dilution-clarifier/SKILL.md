# 估值与稀释澄清器

## R — 原文 (Reading)

> “投资前估值”
>
> 来源: Brad Feld、Jason Mendelson《风险投资交易》第4章。

## I — 方法论骨架 (Interpretation)

这个 skill 把“估值高不高”拆成融资后 cap table。Term Sheet 里的价格通常围绕投资前估值、投资后估值和完全稀释股本展开, 但真实稀释还取决于期权池是否在投前扩充、可转债或 SAFE 如何转换、是否存在未行权期权和保留股份。

书中特别提醒, 期权池放在投前还是投后, 会改变创始人真实承担的稀释。看起来一样的投后占比, 可能对应完全不同的创始人剩余比例。

本 skill 的目标是把每一句估值话术翻译为一张融资前后 cap table, 再连接到控制权和下一轮融资空间。

## A1 — 书中的应用 (Past Application)

### 案例 1: 投前与投后估值

- **问题**: 创业者听到“投后估值 1 亿”就以为公司现在值 1 亿。
- **方法论的使用**: 区分投资前价值和投资后价值。
- **结论**: 投后估值包含新进资金。
- **结果**: 投资人占比应从投资金额和投后估值推导。

### 案例 2: 期权池的隐性稀释

- **问题**: 投资人要求融资前建立 15% 期权池。
- **方法论的使用**: 把期权池纳入完全稀释股本。
- **结论**: 若期权池在投前扩充, 稀释主要由老股东承担。
- **结果**: 估值谈判必须同时谈期权池大小和归属。

### 案例 3: 完全稀释口径

- **问题**: 各方用不同股本口径计算占比。
- **方法论的使用**: 统一已发行股份、期权、认股权、可转换证券等口径。
- **结论**: 没有统一分母, 占比讨论会失真。
- **结果**: 谈判前先固定 cap table 口径。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户问投资前/投资后估值如何计算。
2. 用户想知道融资后创始人、员工、投资人各占多少。
3. 用户收到投资人要求扩大期权池。
4. 用户不确定 SAFE、可转债或未行权期权是否要算进分母。

### 语言信号

- “这轮会稀释多少?”
- “投前投后估值怎么区分?”
- “期权池算谁的稀释?”
- “完全稀释是什么意思?”

### 与相邻 skill 的区分

- 与 `term-sheet-economics-control-audit` 的区别: 本 skill 聚焦价格和稀释; 后者还审清算、反稀释、保护性条款等。
- 与 `founder-control-threshold-map` 的区别: 本 skill 计算比例; 后者判断比例背后的权力。

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **统一输入口径**
   - 完成标准: 收集投资金额、投前/投后估值、现有股份、期权池、可转换证券和新增期权池要求。

2. **重建融资前 cap table**
   - 完成标准: 写出普通股、优先股、期权、未转换工具的数量和百分比。

3. **计算融资后占比**
   - 完成标准: 分别按基础口径和完全稀释口径计算投资人、创始人、员工池和其他股东占比。

4. **标出隐性稀释**
   - 完成标准: 单独说明投前期权池扩充、可转工具折扣/估值上限、未分配期权池对老股东的影响。

5. **连接控制权和下一轮**
   - 完成标准: 提醒稀释后是否触发控制权阈值变化, 以及是否还留有下一轮融资空间。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要求判断公司值多少钱或预测市场估值倍数。
- 用户需要审查清算优先权、反稀释、领售或保护性条款。
- 用户要求税务或证券合规结论。

### 作者在书中警告的失败模式

- 只看估值标题, 不看完全稀释分母。
- 忽视期权池在投前扩充造成的真实稀释。
- 把投后估值误解为融资前公司价值。

### 作者的盲点 / 时代局限

- SAFE、可转债和各地融资工具的流行口径会变化。
- 估值倍数和市场情绪必须用当前市场数据另行验证。

### 容易混淆的邻近方法论

- “公司估价模型”: 本 skill 不做 DCF 或可比公司估值。
- “控制权设计”: 本 skill 输出比例, 不直接设计治理条款。
