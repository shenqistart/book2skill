# 天使投资组合纪律

## R — 原文 (Reading)

> “20~80家公司”
>
> 来源: David S. Rose《超级天使投资》第1章。

## I — 方法论骨架 (Interpretation)

这个 skill 把“这个项目能不能投”改写成“我的组合是否承受得起早期投资的失败率”。书中用天使投资的回报分布说明, 单个项目极不确定, 组合纪律比单点判断更重要。大量项目会失败或只回本, 少数项目承担大部分回报。

因此, 天使投资前应先设总资产上限、单笔上限、组合数量目标、筛选漏斗、跟投预留和退出预期。它不是选股或项目推荐工具, 而是防止个人投资者把偶然热情误当成投资策略。

## A1 — 书中的应用 (Past Application)

### 案例 1: 组合数量降低单点风险

- **问题**: 投资人想把大部分资金投给一个熟人创业项目。
- **方法论的使用**: 用早期投资失败率检查集中风险。
- **结论**: 单项目成功概率不足以支撑重仓。
- **结果**: 先设组合规模和单笔比例。

### 案例 2: 总资产上限

- **问题**: 个人投资者被高回报故事吸引, 忽略流动性。
- **方法论的使用**: 将天使投资限定在可承受损失的资产比例内。
- **结论**: 天使投资应是高风险资产的一小部分。
- **结果**: 先判断自己是否适合做天使。

### 案例 3: 筛选漏斗和跟投预留

- **问题**: 投资人早期投完预算, 后续好项目或跟投无钱。
- **方法论的使用**: 预留跟投资金和时间窗口。
- **结论**: 投资纪律包括不投、少投和留子弹。
- **结果**: 组合管理优先于单笔兴奋。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户想做天使投资, 问第一笔投多少。
2. 用户被朋友项目、热门赛道或明星创始人吸引。
3. 用户想理解天使投资为何需要组合。
4. 用户已经投了几个项目, 想建立跟投和退出纪律。

### 语言信号

- “我想做天使投资。”
- “这个朋友的项目能不能投?”
- “单笔投多少合适?”
- “天使投资要投多少家公司才分散?”

### 与相邻 skill 的区分

- 与 `investor-fit-signal-filter` 的区别: 本 skill 站在投资人资产和组合角度; 后者站在创业者选择投资人角度。
- 与 `financing-readiness-runway` 的区别: 本 skill 判断投资纪律; 后者判断创业公司融资准备。

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **确认适格和风险承受**
   - 完成标准: 明确用户是否能承受本金长期锁定和大比例亏损; 不能确认时不进入项目判断。

2. **设总预算和单笔上限**
   - 完成标准: 将总预算限定为可承受损失部分, 并设单项目最大投入。

3. **设计组合目标**
   - 完成标准: 写出目标项目数、阶段分布、行业集中度和时间跨度。

4. **建立筛选漏斗**
   - 完成标准: 定义来源、初筛、尽调、条款、投后和退出跟踪流程。

5. **输出纪律清单**
   - 完成标准: 给出不投条件、少投条件、跟投条件和复盘周期。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要求推荐具体创业项目或保证收益。
- 用户不具备高风险承受能力, 或把生活资金用于天使投资。
- 用户需要证券、基金募集或合格投资人法律判断。

### 作者在书中警告的失败模式

- 把天使投资当成低风险理财。
- 单项目下注过重。
- 只看故事和关系, 没有组合和尽调纪律。

### 作者的盲点 / 时代局限

- 早期投资数据、退出环境和法律门槛会随市场变化。
- 个人投资者所在司法辖区可能有不同适格要求。

### 容易混淆的邻近方法论

- “创业者融资”: 本 skill 是投资侧, 不用于帮创业者写 BP。
- “财务规划”: 本 skill 只讨论天使投资这一类高风险资产。
