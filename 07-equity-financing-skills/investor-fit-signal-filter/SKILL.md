# 投资人匹配信号过滤器

## R — 原文 (Reading)

> “不要拿它的钱”
>
> 来源: 史蒂文·霍夫曼《穿越寒冬》第二章。

## I — 方法论骨架 (Interpretation)

这个 skill 把“谁愿意投我”改写成“谁的资金逻辑、回报压力和战略利益与我匹配”。不同投资人的钱不是同质商品: 天使看人和早期机会, 种子基金看验证速度, VC 看基金规模和高倍回报, 企业投资人可能带来资源也可能带来客户冲突, 加速器可能提供网络但会标准化拿股。

筛投资人要同时看五类信号: 阶段匹配、单笔金额匹配、退出回报匹配、资源和决策速度匹配、利益冲突。书中尤其提醒, 企业投资人的战略关系可能让其他客户或未来买家产生顾虑。

## A1 — 书中的应用 (Past Application)

### 案例 1: 大基金不适合小结果

- **问题**: 创业者追逐最大牌 VC, 但项目可能只能成长为中型公司。
- **方法论的使用**: 检查基金规模与回报要求。
- **结论**: 大基金需要足够大的退出结果, 否则即使投资也可能后续支持不足。
- **结果**: 目标投资人应和公司天花板匹配。

### 案例 2: 企业投资人的战略冲突

- **问题**: 一家企业投资人愿意投资, 但它是潜在客户的竞争对手。
- **方法论的使用**: 检查客户、渠道、未来融资和退出方是否会产生顾虑。
- **结论**: 战略资金可能带来资源, 也可能关闭其他市场机会。
- **结果**: 不能只看钱和品牌背书。

### 案例 3: 创业者也要尽调投资人

- **问题**: 投资人愿意签 TS, 创业者没有反向了解对方。
- **方法论的使用**: 通过被投公司、退出记录、决策流程和增值能力反向尽调。
- **结论**: 投资人是长期股东, 也应被选择。
- **结果**: 融资从单向求钱变成双向匹配。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户有多个投资人意向, 不知道如何排序。
2. 用户想知道天使、VC、企业投资人、加速器哪类更合适。
3. 用户担心战略投资人影响客户或下一轮融资。
4. 用户想做投资人名单, 但缺少筛选标准。

### 语言信号

- “应该找哪类投资人?”
- “这个战略投资人能不能拿?”
- “大基金是不是更好?”
- “投资人除了钱还要看什么?”

### 与相邻 skill 的区分

- 与 `financing-readiness-runway` 的区别: 本 skill 假设公司已决定进入融资筛选; 前者判断是否该融资。
- 与 `term-sheet-economics-control-audit` 的区别: 本 skill 筛人; 后者审条款。

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **定义融资画像**
   - 完成标准: 写明公司阶段、资金需求、行业、增长速度、退出想象和下一轮目标。

2. **划分投资人类型**
   - 完成标准: 将候选投资人归入天使、种子、VC、企业投资人、加速器、产业资本或其他类别。

3. **检查五类匹配**
   - 完成标准: 对每位投资人打标阶段、金额、回报模型、资源/决策、冲突五项。

4. **做反向尽调问题表**
   - 完成标准: 至少列出被投公司反馈、决策周期、后续轮支持、冲突案例和退出偏好问题。

5. **输出优先级**
   - 完成标准: 给出优先接触、谨慎接触、暂不接触三类, 并说明最关键原因。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要求最新投资机构名单、联系人、融资市场实时热度。
- 用户要求保证拿到某机构投资。
- 用户已经进入 Term Sheet 细节谈判。

### 作者在书中警告的失败模式

- 把所有投资人的钱看成一样。
- 只追品牌, 不看基金规模和回报压力。
- 忽略企业投资人带来的客户和战略冲突。

### 作者的盲点 / 时代局限

- 投资机构策略、基金状态和合伙人偏好变化快, 需要实时尽调。
- 不同国家和行业的融资生态差异很大。

### 容易混淆的邻近方法论

- “融资话术”: 本 skill 不负责包装叙事。
- “投资建议”: 本 skill 不推荐具体机构或投资项目。
