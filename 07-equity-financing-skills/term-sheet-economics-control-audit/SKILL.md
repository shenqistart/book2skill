# Term Sheet 经济与控制条款审计

## R — 原文 (Reading)

> “交易结构”
>
> 来源: Brad Feld、Jason Mendelson《风险投资交易》第3章。

## I — 方法论骨架 (Interpretation)

这个 skill 把 Term Sheet 分成两组条款: 经济因素和控制因素。经济因素回答“钱怎么分、下行保护如何影响收益”, 包括估值、投资金额、期权池、清算优先权、反稀释、股息、赎回权等。控制因素回答“谁能决定什么”, 包括董事会席位、保护性条款、投票权、领售/跟售、信息权、优先购买权、创始人 vesting 和竞业限制等。

《创业之初你不可不知的融资知识》提醒, 签 Term Sheet 并不等于钱到账。之后还有尽调、投资委员会、正式文件和交割条件。很多创业者不理解条款含义, 只关注估值, 最后在清算、反稀释或董事会权力上付出更高代价。

本 skill 的目标是先把条款翻译成人话, 再标出哪些条款需要谈判、哪些需要律师复核、哪些可能导致未来股权战争。

## A1 — 书中的应用 (Past Application)

### 案例 1: 只看估值忽视清算优先权

- **问题**: 创业者接受较高估值, 但没有理解多倍清算优先权。
- **方法论的使用**: 把退出收益按不同退出金额情景分配。
- **结论**: 高估值可能被强保护条款抵消。
- **结果**: 经济条款要用情景表测试。

### 案例 2: 保护性条款改变控制权

- **问题**: 投资人股权比例不高, 但要求广泛 veto。
- **方法论的使用**: 把 veto 事项列为控制权清单。
- **结论**: 小比例股东也可能对融资、预算、并购、人员任免形成阻断。
- **结果**: 条款审计必须和控制权地图连接。

### 案例 3: TS 后仍可能失败

- **问题**: 创业者签完 TS 后停止寻找备选方案。
- **方法论的使用**: 把尽调、投委会、正式文件和交割条件列入完成路径。
- **结论**: TS 是里程碑, 不是到账。
- **结果**: 公司仍需管理 runway 和备选融资路径。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户收到 Term Sheet, 想知道哪些条款危险。
2. 用户只看估值, 忽略清算优先权、反稀释和董事会条款。
3. 用户需要把 TS 翻译给创始团队或董事会。
4. 用户想判断签 TS 后还有哪些交割风险。

### 语言信号

- “这份 TS 怎么看?”
- “清算优先权是什么意思?”
- “保护性条款会不会太多?”
- “签了意向书是不是钱就确定了?”

### 与相邻 skill 的区分

- 与 `valuation-dilution-clarifier` 的区别: 本 skill 覆盖完整 TS; 估值和稀释可先交给后者精算。
- 与 `equity-conflict-pre-mortem` 的区别: 本 skill 审当前条款; 后者预演未来冲突。

## E — 可执行步骤 (Execution)

当 skill 被激活后, agent 应按以下步骤执行:

1. **拆条款清单**
   - 完成标准: 将条款分为经济因素、控制因素、创始人义务、交割条件和杂项。

2. **翻译经济后果**
   - 完成标准: 对估值、期权池、清算优先权、反稀释、股息和赎回权做普通话解释, 并说明对不同退出金额的影响。

3. **翻译控制后果**
   - 完成标准: 对董事会、保护性条款、投票权、领售/跟售、信息权和优先权标出谁获得通过权或阻断权。

4. **列交割风险**
   - 完成标准: 写明尽调、投委会、正式协议、先决条件、排他期和违约责任。

5. **输出谈判优先级**
   - 完成标准: 标出红线条款、可谈条款、可接受但需披露条款、必须律师复核条款。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要求代替律师给出法律结论或直接签署建议。
- 用户只问基础投前/投后估值计算, 应先用 `valuation-dilution-clarifier`。
- 用户要求隐瞒负面信息或规避尽调。

### 作者在书中警告的失败模式

- 把 TS 当成最终到账。
- 只看估值, 不看经济保护和控制权条款。
- 不理解反稀释、清算优先权、领售和创始人 vesting 的长期影响。

### 作者的盲点 / 时代局限

- 条款市场惯例会随融资周期变化, 需要结合当前轮次、地区和行业复核。
- 正式协议文本可能比 TS 更复杂, 必须交给专业律师审阅。

### 容易混淆的邻近方法论

- “融资叙事”: 本 skill 不写 BP, 只审条款。
- “股东冲突案例”: 本 skill 可发现冲突种子, 但完整预演交给 `equity-conflict-pre-mortem`。
