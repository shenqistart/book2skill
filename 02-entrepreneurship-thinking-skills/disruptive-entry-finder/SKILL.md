# 破坏式入场点识别器

## R — 原文 (Reading)

> “客户是在雇用产品去完成某项特别的任务。”
>
> “新市场破坏策略是与零消费竞争。”
>
> “低端市场破坏策略关注被过度服务的客户。”
>
> 来源：《创新者的解答》。

## I — 方法论骨架 (Interpretation)

这个 skill 把“怎么打过大公司”改写成“从大公司不愿意或不适合服务的任务进入”。

第一，先问客户要完成什么任务，而不是先问产品属于哪个品类。
第二，新市场破坏来自非消费场景：用户过去没有能力、渠道或预算完成这个任务。
第三，低端破坏来自过度服务场景：主流产品性能超过部分客户需求，创业公司可以用更简单、便宜、方便的方案切入。
第四，破坏式路径成立，需要既有玩家因利润率、组织流程、渠道或品牌定位而缺少反击动机。

它适合帮助创业者避开正面对抗，把机会从“大公司会不会做”细化为“大公司为什么不愿意这样做”。

## A1 — 书中的应用 (Past Application)

### 案例 1：按客户任务重新看市场

- **问题**：产品类别不能解释用户为什么购买。
- **方法论的使用**：观察用户在具体情境中雇用产品完成什么任务。
- **结论**：真正竞争对手可能不是同品类产品，而是完成同一任务的替代方案。
- **结果**：入场点从品类竞争转为任务竞争。

### 案例 2：索尼早期的小型化产品

- **问题**：新产品如何避开主流大厂的性能竞赛。
- **方法论的使用**：用便携、低成本和新场景服务原本不能消费的人群。
- **结论**：新市场破坏不从主流性能指标开始，而从可及性开始。
- **结果**：非消费场景成为增长入口。

### 案例 3：小米 MIUI 的社区入口

- **问题**：进入硬件强竞争行业前如何建立立足点。
- **方法论的使用**：先服务一群对系统体验敏感、愿意反馈的早期用户。
- **结论**：软件体验和社区关系构成进入硬件市场的前置路径。
- **结果**：后续可转入 `validated-learning-loop` 验证产品假设。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户认为“大公司太强，我们没机会”。
2. 用户想进入成熟行业，但缺少低端、新市场或任务切入点。
3. 用户怀疑产品是否只是普通差异化，而非破坏式路径。
4. 用户需要判断目标客户是否被过度服务或尚未被服务。

### 语言信号

- “怎么从大厂缝隙里切入？”
- “这个市场有没有非消费人群？”
- “低配、低价、简单方案会不会有机会？”
- “客户真正想完成的任务是什么？”

### 与相邻 skill 的区分

- 与 `nonconsensus-monopoly-thesis` 的区别：后者判断机会假说；本 skill 找入场路径。
- 与 `validated-learning-loop` 的区别：本 skill 输出破坏路径假设；后者设计实验验证。
- 与 `user-frontline-flywheel` 的关系：入场后需要前线反馈持续校正客户任务。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **定义客户任务**
   - 完成标准：写出用户在什么场景下、为了什么进展而使用或替代某个产品。

2. **识别现有方案的过度/不足服务**
   - 完成标准：说明主流产品在哪些性能、价格、复杂度或渠道上超过或低于目标客户需要。

3. **判断破坏类型**
   - 完成标准：给出新市场破坏、低端破坏、延续性创新或非破坏式差异化四类判断。

4. **检查既有玩家动机**
   - 完成标准：说明大公司为什么短期不愿意或难以服务这个入口。

5. **设计入场假设**
   - 完成标准：给出初始用户、最小产品、关键指标和下一步验证方式。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只是想做广告、品牌文案或包装差异化。
- 竞争优势来自监管牌照、资本补贴或独家资源，而不是客户任务和入场路径。
- 目标市场已明确，当前问题是组织管理或现金危机。

### 作者在书中警告的失败模式

- 把所有新产品都叫“颠覆”。
- 用主流性能指标衡量早期破坏式产品，提前误判。
- 忽略既有公司的资源配置和利润动机。

### 作者的盲点 / 时代局限

- 破坏式创新理论并不保证每个低端入口都能上升。
- 平台经济和生态锁定可能让小入口更难扩张。
- 强监管行业必须先处理合规和风险承受能力。

### 容易混淆的邻近方法论

- “细分市场定位”：本 skill 要求说明任务、非消费/过度服务和不对称动机。
- “低价策略”：低价只有服务破坏路径时才是方法，不是目的。
