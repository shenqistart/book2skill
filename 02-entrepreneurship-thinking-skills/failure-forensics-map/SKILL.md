# 创业失败链条复盘

## R — 原文 (Reading)

> “这些失败基因，使他们始终无法真正地超越自我。”
>
> 来源：吴晓波《大败局》序言。

## I — 方法论骨架 (Interpretation)

创业失败很少只因为一个人蠢、一个功能错或一次融资失败。

这个 skill 把失败看成链条：最初的机会判断是否错了，商业模式是否存在悖论，资本是否放大了错误，创始人是否被英雄主义绑架，组织是否无法传递坏消息，现金和交付是否最终断裂。

它的目标不是审判，而是找出可预防、可监控、可修复的结构性原因。

## A1 — 书中的应用 (Past Application)

### 案例 1：《大败局》的失败基因

- **问题**：为什么一批曾经辉煌的中国企业迅速崩塌。
- **方法论的使用**：吴晓波在多个案例中寻找共同失败基因，而不是只讲单个事故。
- **结论**：激情、扩张、制度缺陷、道德风险和管理短板会互相强化。
- **结果**：失败案例成为后来企业的反向教材。

### 案例 2：《十亿美金的教训》的英雄主义陷阱

- **问题**：互联网创业公司为什么从英雄故事走向失败。
- **方法论的使用**：作者追踪创始人英雄主义、资本诱惑、用户体验细节和商业领袖转型失败。
- **结论**：个人英雄如果不能升级为商业领导，会反过来拖垮公司。
- **结果**：形成互联网失败案例的结构化复盘。

### 案例 3：《创业之殇》的初创失败原因

- **问题**：初创企业为什么反复死在相似问题上。
- **方法论的使用**：按市场、产品、营销、执行和资金流动拆解失败。
- **结论**：失败通常是多个基本问题未被及时验证和修正。
- **结果**：给早期团队提供事前检查框架。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 项目已经失败或接近失败，需要复盘。
2. 一个明星项目增长很快，但出现组织、现金或模式异常。
3. 团队总把问题归咎于某个人，无法看到结构链条。
4. 创始人想从失败案例中提炼预警指标。

### 语言信号

- “这次失败到底因为什么？”
- “我们是不是在重演某种败局？”
- “这个项目增长很快但哪里不对劲。”
- “怎么做创业复盘？”

### 与相邻 skill 的区分

- 与 `hard-things-wartime-leadership` 的区别：后者处理正在发生的危机；本 skill 做复盘和预警。
- 与 `lean-business-model-risk-map` 的区别：后者事前检查模型；本 skill 用失败链条反向诊断。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **画出失败时间线**
   - 完成标准：列出关键决策、外部变化、指标恶化、组织反应和最终结果。

2. **拆六类失败因子**
   - 完成标准：分别检查市场错判、模式悖论、资本冒进、创始人英雄主义、组织失灵和现金断裂。

3. **识别因果链条**
   - 完成标准：说明哪些因素是根因，哪些是放大器，哪些只是表面症状。

4. **对照反事实**
   - 完成标准：提出 2-3 个当时可采取的不同动作，并说明需要什么信号才能触发。

5. **生成预警清单**
   - 完成标准：列出未来可监控的指标、会议问题、用户信号和现金阈值。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 公司还在生死危机中，需要马上裁剪、沟通和止血，应优先用 `hard-things-wartime-leadership`。
- 用户只想找替罪羊或写对外甩锅声明。
- 涉及法律责任、审计、破产或投资纠纷，需要专业意见。

### 作者在书中警告的失败模式

- 激情、融资和媒体声量掩盖商业基本面。
- 创始人无法从个人英雄升级为商业领袖。
- 模式看上去很美，但供应、库存、渠道、现金或治理不闭合。

### 作者的盲点 / 时代局限

- 失败叙事存在事后偏差，不能把所有结果都解释成必然。
- 中国 1990-2010 年代商业环境与当前技术、监管和资本环境不同。

### 容易混淆的邻近方法论

- 复盘会议：本 skill 是结构化失败取证，不是普通总结。
- 风险管理：本 skill 从真实失败案例反推预警，不替代专业风控。
