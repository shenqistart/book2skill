# 管理者产出杠杆

## R — 原文 (Reading)

> “经理人的产出=他直接管辖部门的产出+他间接影响所及部门的产出。”
>
> 来源：安迪·格鲁夫《格鲁夫给经理人的第一课》。

## I — 方法论骨架 (Interpretation)

管理者的价值不等于自己完成了多少任务，而是让多少人的产出变得更好。

这个 skill 用“产出杠杆”重排管理者的时间：哪些会议能减少误解，哪些一对一能发现问题，哪些培训能复制能力，哪些决策能解除阻塞，哪些授权能让团队更快行动。

它尤其适合从创始人个人英雄阶段进入组织扩张阶段的公司。

## A1 — 书中的应用 (Past Application)

### 案例 1：格鲁夫的经理人产出公式

- **问题**：如何衡量经理人的真实贡献。
- **方法论的使用**：把经理人产出定义为直接和间接影响的团队产出。
- **结论**：管理者应优先做能放大多人产出的活动。
- **结果**：会议、培训、一对一和决策被重新理解为杠杆工具。

### 案例 2：关明生区分效率和效力

- **问题**：中小企业管理容易忙于局部效率，却忽略方向和组织战斗力。
- **方法论的使用**：先判断是否在做正确的事，再提升执行效率。
- **结论**：管理杠杆要服务于正确目标。
- **结果**：避免“很忙但不产出”的管理状态。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 创始人每天很忙，但团队依然等待他拍板。
2. 经理人不知道会议、一对一、培训和授权怎么取舍。
3. 团队跨部门协作低效，信息传递失真。
4. 管理者仍用个人贡献者方式工作。

### 语言信号

- “我太忙了，但团队产出没变好。”
- “哪些管理动作最有杠杆？”
- “会议到底该不该开？”
- “团队什么事都等我决策。”

### 与相邻 skill 的区分

- 与 `talent-culture-operating-system` 的区别：本 skill 优化管理活动；后者设计人才和文化机制。
- 与 `hard-things-wartime-leadership` 的区别：本 skill 适合常态管理；后者适合危机战时。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **定义团队产出**
   - 完成标准：写出团队的核心产出指标、当前瓶颈和被影响对象。

2. **列出管理活动**
   - 完成标准：列出现有会议、一对一、培训、决策、招聘、文档和跨团队沟通。

3. **评估杠杆率**
   - 完成标准：按影响人数、影响持续时间、是否解除瓶颈、是否复制能力评估每项活动。

4. **重排管理日程**
   - 完成标准：删除低杠杆活动，保留或增强高杠杆活动，并设置节奏。

5. **建立反馈指标**
   - 完成标准：用团队产出、决策周期、返工率、阻塞时间和人才成长验证管理效果。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只是个人时间管理、番茄钟或待办清单问题。
- 公司处在裁员、现金危机或重大转型中，应转 `hard-things-wartime-leadership`。
- 团队根本没有清晰目标，需要先回到战略或商业模式。

### 作者在书中警告的失败模式

- 管理者沉迷个人执行，团队能力没有提升。
- 会议没有目的，成为信息噪音。
- 对下属问题只批评不培训，导致产出不稳定。

### 作者的盲点 / 时代局限

- 工厂和半导体管理经验需要结合现代远程、异步和创造型团队调整。
- 过度量化管理可能压制探索型工作。

### 容易混淆的邻近方法论

- 个人效率：本 skill 的单位是团队产出，不是个人忙碌。
- OKR：OKR 定义目标，本 skill 设计管理杠杆。
