# 战时艰难决策协议

## R — 原文 (Reading)

> “真正的难题并不是设置一个宏伟目标，而是你在没有实现目标时不得不忍痛裁员。”
>
> 来源：Ben Horowitz《创业维艰》前言。

## I — 方法论骨架 (Interpretation)

创业的困难不是讲战略时最明显，而是在坏消息已经发生、没有好选项、任何决定都会伤人时最明显。

这个 skill 要求领导者先承认现实：现金、客户、产品、组织和士气中哪一个已经进入危险区。然后把模糊恐惧变成可行动问题，决定谁需要知道什么、何时沟通、先保什么、砍什么、谁负责下一步。

战时领导不是强势表演，而是承担事实、决策和后果。

## A1 — 书中的应用 (Past Application)

### 案例 1：霍洛维茨处理公司危机

- **问题**：公司在市场和现金压力下必须做痛苦决策。
- **方法论的使用**：不回避坏消息，直接处理裁员、客户、融资和组织士气。
- **结论**：领导者不能把困难外包给愿景或流程。
- **结果**：形成面向真实困境的 CEO 行动原则。

### 案例 2：小米低谷复盘

- **问题**：小米高速增长后遇到放缓和低谷。
- **方法论的使用**：雷军复盘早期成功、中期问题和重回增长的原因。
- **结论**：低谷不是只靠鼓劲，而要回到能力、模式和组织调整。
- **结果**：低谷经验被纳入创业方法论。

### 案例 3：优衣库海外失败后的修正

- **问题**：伦敦等海外扩张失败。
- **方法论的使用**：柳井正分析败因，把失败转为下一次实践参考。
- **结论**：危机要被复盘和制度化，不能只归咎外部环境。
- **结果**：组织继续调整全球化方法。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 公司现金只够几个月，需要裁剪业务或团队。
2. 创始人必须裁员、降薪、关闭产品线或放弃客户。
3. 坏消息已经存在，但团队还在互相安慰。
4. 常态管理方法失效，需要战时优先级。

### 语言信号

- “现金只够 X 个月了。”
- “要不要裁员？”
- “坏消息怎么跟团队说？”
- “现在没有好选项，怎么选？”

### 与相邻 skill 的区分

- 与 `manager-output-leverage` 的区别：后者是常态管理杠杆；本 skill 是危机下的决策协议。
- 与 `failure-forensics-map` 的区别：本 skill 面向正在发生的困难；后者面向复盘和预防重演。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **确认战时事实**
   - 完成标准：列出现实约束，包括现金 runway、客户风险、产品风险、团队风险和时间窗口。

2. **区分可保与必须砍**
   - 完成标准：明确公司必须保住的核心资产，以及应该停止、裁剪或延后的项目。

3. **设计决策顺序**
   - 完成标准：按现金影响、客户影响、组织影响和不可逆性排序行动。

4. **准备沟通**
   - 完成标准：分别写出对核心团队、全员、客户和投资人的事实口径、承诺边界和下一步。

5. **建立 7-30 天战时节奏**
   - 完成标准：定义每日/每周指标、负责人、复盘节奏和何时转入失败复盘。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 公司处在平稳期，只是想优化会议、绩效或组织结构。
- 用户需要劳动法、融资条款、破产清算等专业法律/财务意见。
- 用户只是想写励志动员稿，不愿面对具体事实。

### 作者在书中警告的失败模式

- 领导者隐瞒坏消息，导致组织失去信任。
- 裁员拖泥带水，反复伤害士气。
- 用愿景替代现金和客户现实。

### 作者的盲点 / 时代局限

- 硅谷 VC 支持公司的危机模式不完全适用于自举、中小企业或中国监管环境。
- 战时强决策可能压制必要的反对意见，需要保留事实通道。

### 容易混淆的邻近方法论

- 危机公关：本 skill 先解决公司真实行动，再处理表达。
- 失败复盘：本 skill 处理进行中的困难，不是事后总结。
