# 用户前线经营飞轮

## R — 原文 (Reading)

> “做全球最好的手机，只卖一半的价钱，让每个人都能买得起。”
>
> 来源：雷军《小米创业思考》第一章。

## I — 方法论骨架 (Interpretation)

用户不是调研表里的画像，而是在现场不断用选择、抱怨、复购、推荐和沉默投票的人。

这个 skill 把用户现场变成经营飞轮：先接近真实用户，再把反馈变成产品和服务改进；改进带来口碑和复购；规模提升后继续用成本效率回馈用户；前线再把新问题传回总部。

它适合那些容易被总部幻觉、投放数据或品牌叙事带偏的业务。

## A1 — 书中的应用 (Past Application)

### 案例 1：小米和用户交朋友

- **问题**：小米如何在资源不足时建立早期口碑。
- **方法论的使用**：通过 MIUI、米粉社区和用户反馈，让用户参与产品改进。
- **结论**：用户不仅是购买者，也是产品共创和传播节点。
- **结果**：早期社区成为小米产品和品牌飞轮的一部分。

### 案例 2：沃尔玛顾客第一和前线巡视

- **问题**：低价零售如何持续运转。
- **方法论的使用**：萨姆·沃尔顿重视门店、同事、顾客反馈和节俭运营。
- **结论**：成本效率必须回到顾客价值，而不是只变成公司利润。
- **结果**：顾客第一、低价和组织文化形成增强回路。

### 案例 3：优衣库从门店失败中修正

- **问题**：门店扩张和商品规划多次失败。
- **方法论的使用**：柳井正把失败门店、商品、店长能力和供应链经验持续复盘。
- **结论**：前线失败是下一轮经营设计的输入。
- **结果**：优衣库形成以商品、门店和组织学习为核心的改进机制。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户有增长或复购问题，但不清楚一线体验哪里断了。
2. 团队过度依赖投放、GMV 或下载量，缺少真实用户反馈。
3. 零售、社区、服务或消费产品需要建立口碑飞轮。
4. 总部策略和前线反馈冲突。

### 语言信号

- “用户为什么不复购？”
- “我们离真实用户太远了。”
- “门店反馈和总部判断不一致。”
- “怎么把口碑做成飞轮？”

### 与相邻 skill 的区分

- 与 `validated-learning-loop` 的区别：后者做单次验证；本 skill 建立持续经营飞轮。
- 与 `manager-output-leverage` 的区别：本 skill 从用户现场出发；后者从管理者产出出发。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **定位真实现场**
   - 完成标准：明确用户在哪里产生选择、抱怨、复购、推荐或流失，而不是只看报表。

2. **收集前线信号**
   - 完成标准：列出至少 5 类信号，包括购买、使用、退货、客服、门店/社群反馈和沉默流失。

3. **找到飞轮断点**
   - 完成标准：判断断点在产品价值、服务体验、价格成本、渠道承诺、复购理由或口碑传播。

4. **设计改进回路**
   - 完成标准：说明前线信号如何进入产品、运营、供应或培训，并设定回访指标。

5. **输出飞轮图**
   - 完成标准：用“用户现场 -> 改进 -> 口碑/复购 -> 成本效率 -> 用户价值”的链条描述下一步动作。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只是要投放账户优化、广告素材或短期活动方案。
- 业务没有可接触的真实用户现场，需要先做基础验证。
- 问题核心是组织结构、绩效或管理者杠杆，应转 `manager-output-leverage`。

### 作者在书中警告的失败模式

- 总部用战略想象替代门店和用户现场。
- 只把低价当营销，不把成本效率回馈用户。
- 把社区当流量池，而不是产品共创和信任关系。

### 作者的盲点 / 时代局限

- 零售和消费品经验不能直接套用到低频 B2B 或强监管业务。
- 用户反馈不等于用户永远正确，需要和商业模式、成本和长期方向一起判断。

### 容易混淆的邻近方法论

- 用户调研：本 skill 更强调持续运营回路。
- 增长飞轮：本 skill 要求飞轮必须回到真实用户价值，而不只是流量循环。
