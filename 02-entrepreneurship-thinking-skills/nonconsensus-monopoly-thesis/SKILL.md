# 非共识秘密和可防守垄断假说

## R — 原文 (Reading)

> “在什么重要问题上你与其他人有不同看法？”
>
> “竞争性市场很难赚钱。”
>
> “每个初创公司都应该在非常小的市场内起步。”
>
> 来源：《从 0 到 1》。

## I — 方法论骨架 (Interpretation)

这个 skill 把“机会大不大”改写成“是否存在非共识且能防守的独特供给”。

第一，好机会通常不是所有人都同意的共识，而是重要、真实、尚未被充分相信的秘密。
第二，非共识本身不够，必须能转化为可捕获价值的市场结构。
第三，创业公司不应一开始定义巨大市场，而应先找到可占据的小市场，并在其中建立强地位。
第四，可防守优势常来自专有技术、网络效应、规模经济、品牌、分发或独特组织能力。

它适合在投入验证前先清理战略叙事：用户是否真的需要，市场是否定义清楚，为什么不是大公司顺手做掉。

## A1 — 书中的应用 (Past Application)

### 案例 1：《从 0 到 1》的小市场起步

- **问题**：创业公司如何避免在大市场中被竞争吞噬。
- **方法论的使用**：先占据一个很小但真实的市场，再逐步扩大边界。
- **结论**：一开始讲总市场巨大，常常掩盖没有立足点。
- **结果**：机会判断从规模叙事转向进入顺序和防守结构。

### 案例 2：小米从 MIUI 与早期用户切入

- **问题**：没有完整硬件经验时如何进入强竞争行业。
- **方法论的使用**：先在软件体验和核心用户社区中建立独特关系，再进入更大的硬件市场。
- **结论**：非共识不是“我要做手机”，而是软件、硬件、互联网结合能改写用户体验。
- **结果**：小市场切入让后续扩张有了用户和能力基础。

### 案例 3：创新者理论补充入场路径

- **问题**：非共识机会如何避开既有强者。
- **方法论的使用**：用新市场或低端市场寻找大公司动机不足的入口。
- **结论**：可防守机会常来自不对称动机，而非正面对抗。
- **结果**：机会假说进入 `disruptive-entry-finder` 做更细判断。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户说“这个赛道很大”，但没有说明独特立足点。
2. 用户想判断创业想法是否只是功能拼装或跟风。
3. 用户需要找小市场、早期用户和可扩张顺序。
4. 用户在融资叙事中夸大市场，但忽略竞争和防守。

### 语言信号

- “这个机会是不是红海？”
- “我们有什么别人没看到的洞察？”
- “怎么找到从 0 到 1 的切入点？”
- “大公司会不会很快复制？”

### 与相邻 skill 的区分

- 与 `founder-mission-value-system` 的区别：本 skill 判断机会结构，不判断价值底线。
- 与 `disruptive-entry-finder` 的区别：本 skill 先形成机会假说；后者定位新市场/低端入场点。
- 与 `validated-learning-loop` 的关系：本 skill 输出待验证的战略假设，后者设计 MVP 和指标。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **写出非共识命题**
   - 完成标准：用一句话说明“我们相信但多数人未充分相信的事实”。

2. **检查重要性和真实性**
   - 完成标准：说明该事实对用户价值、成本结构、技术路径或分发方式为何重要。

3. **重新定义小市场**
   - 完成标准：给出一个能先占据的细分用户群、场景或任务，不使用泛泛大市场口径。

4. **识别防守结构**
   - 完成标准：至少检查专有技术、网络效应、规模经济、品牌、渠道或组织能力中的 3 项。

5. **输出机会假说**
   - 完成标准：给出“可验证 / 太共识 / 太小 / 无防守 / 信息不足”，并列出下一步验证问题。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户已经只需要产品排期、工程拆分或 UX 优化。
- 用户要求宏观趋势预测、投资建议或具体估值。
- 机会主要取决于实时政策、牌照或法律解释，必须先补外部信息。

### 作者在书中警告的失败模式

- 把“市场很大”当成机会成立。
- 把差异化话术误当作垄断结构。
- 一开始就和强者正面对抗，没有小市场立足点。

### 作者的盲点 / 时代局限

- 垄断叙事可能低估公共利益、合规和平台治理。
- 非共识容易滑向自我催眠，必须进入真实验证。
- 有些行业不允许或不适合形成强垄断，需要改用效率或服务质量优势。

### 容易混淆的邻近方法论

- “蓝海战略”：本 skill 更强调非共识秘密和可捕获价值。
- “护城河分析”：本 skill 用于早期机会假说，不替代成熟公司竞争分析。
