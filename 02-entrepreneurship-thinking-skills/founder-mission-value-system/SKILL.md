# 创始人使命、用户价值和不做清单系统

## R — 原文 (Reading)

> “做对的事情，把事情做对。”
>
> “好的公司都一定是有一个长长的 Stop Doing List。”
>
> “商业的目的是什么，如何让商业实现最大化的现实意义？我的答案是：效率。”
>
> 来源：《段永平投资问答录》《小米创业思考》。

## I — 方法论骨架 (Interpretation)

这个 skill 把“我要创业”改写成一个经营边界系统。

第一，使命不是标语，而是企业长期存在的理由：为谁创造什么价值，为什么这个价值值得持续投入。
第二，用户价值必须先于增长动作；如果增长损害信任、体验或长期效率，即使短期赚钱也要进入不做清单。
第三，本分不是保守，而是承认能力圈、边界和代价，先排除不该赚的钱、不该碰的客户、不该扩张的赛道。
第四，使命要落到组织动作：产品取舍、人才标准、成本纪律、客户承诺和失败时的沟通方式。

它适合做创业问题的第一道闸门：先判断事情是否值得做，再进入机会、验证、组织或危机 skill。

## A1 — 书中的应用 (Past Application)

### 案例 1：小米把效率作为商业目的

- **问题**：智能硬件公司如何避免只讲配置、渠道和价格战。
- **方法论的使用**：把用户利益、极致效率、口碑和长期关系放在同一个经营系统中。
- **结论**：创业理由不是“进入手机市场”，而是用更高效率把好产品交给用户。
- **结果**：使命约束了小米对产品、价格、渠道和用户关系的判断。

### 案例 2：段永平的本分和不做清单

- **问题**：一个看似赚钱的机会是否应该做。
- **方法论的使用**：先问是否是对的事情，是否在能力圈内，是否会损害长期信誉。
- **结论**：很多正确动作表现为停止错误动作，而不是增加新动作。
- **结果**：把经营判断从短期收益转到长期边界。

### 案例 3：沃尔玛和优衣库的顾客第一

- **问题**：零售企业的经营理念如何变成日常动作。
- **方法论的使用**：把节俭、门店反馈、低价高质和员工责任纳入一套价值约束。
- **结论**：价值观只有能改变成本、服务和现场决策时才有效。
- **结果**：使命不再停留在宣传，而是进入运营节奏。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户问“这个创业方向为什么值得做”“愿景和使命怎么定”。
2. 用户面对高增长但可能损害用户信任、合规或长期品牌的动作。
3. 用户想做很多机会，但缺少不做清单和能力圈边界。
4. 用户的团队文化停留在口号，无法指导产品、招聘和资源取舍。

### 语言信号

- “这件事到底值不值得创业？”
- “使命、愿景、价值观怎么落地？”
- “这个赚钱动作会不会伤用户？”
- “哪些机会应该坚决不做？”

### 与相邻 skill 的区分

- 与 `nonconsensus-monopoly-thesis` 的区别：本 skill 先判断价值边界；后者判断机会结构和垄断潜力。
- 与 `talent-culture-operating-system` 的区别：本 skill 定义价值边界；后者把边界变成组织和人才机制。
- 与 `failure-forensics-map` 的关系：失败复盘常反向揭示使命缺失、边界失守和创始人自我膨胀。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **定义存在理由**
   - 完成标准：写出目标用户、核心价值、长期承诺和不做这件事的社会/用户损失。

2. **检查用户价值**
   - 完成标准：列出用户获得的真实收益、支付的显性/隐性成本、可能被牺牲的信任或体验。

3. **画能力圈和不做清单**
   - 完成标准：列出团队已具备能力、缺失能力、明确不做的客户/渠道/增长动作。

4. **把使命落到经营动作**
   - 完成标准：至少给出产品、定价、获客、招聘、成本或客服中的 3 个具体约束。

5. **输出决策**
   - 完成标准：给出“值得做 / 暂缓 / 不做 / 需要先补证据”，并说明最关键的价值边界。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只要求写品牌口号、愿景文案或宣传稿。
- 用户需要实时融资、法律、税务或证券投资建议。
- 用户已经明确要做 MVP 或商业模式验证，应转 `validated-learning-loop` 或 `lean-business-model-risk-map`。

### 作者在书中警告的失败模式

- 把趋势、融资或短期利润误当作企业存在理由。
- 用“用户第一”包装实际损害用户信任的增长动作。
- 没有不做清单，导致资源被机会主义消耗。

### 作者的盲点 / 时代局限

- 成功企业家的使命叙事可能带有事后解释色彩。
- “长期主义”不能替代现金约束、行业监管和竞争事实。
- 使命和本分需要制度承接，否则容易变成创始人个人偏好。

### 容易混淆的邻近方法论

- “企业文化建设”：本 skill 只处理价值边界，不完整设计文化制度。
- “品牌定位”：本 skill 不输出品牌语义，而是判断经营理由和不做清单。
