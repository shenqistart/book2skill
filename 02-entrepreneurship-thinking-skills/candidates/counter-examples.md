# Counter-example Candidates

## CE01 风口投机替代能力准备

- **source_chapter**: 《小米创业思考》第一章
- **failure_mode**: 只看到风口，不看行业积累、团队能力、模式生命力和持续执行。
- **mechanism**: 外部机会放大短期结果，让创业者误以为趋势等同于护城河。
- **bound_to**: [nonconsensus-monopoly-thesis, founder-mission-value-system]

## CE02 用大市场叙事伪装竞争红海

- **source_chapter**: 《从 0 到 1》第 3 章
- **failure_mode**: 把市场定义得很大来证明机会，却没有任何可防守的独特供给。
- **mechanism**: 非垄断者倾向缩小竞争叙述或夸大独特性，掩盖自己只是红海一员。
- **bound_to**: [nonconsensus-monopoly-thesis]

## CE03 把 MVP 当粗糙产品

- **source_chapter**: 《精益创业》《精益创业实战》
- **failure_mode**: 以“快速试错”为名发布低质量产品，却没有验证明确假设。
- **mechanism**: 团队只节省开发成本，没有定义学习目标和判停指标。
- **bound_to**: [validated-learning-loop]

## CE04 英雄主义压倒商业领导

- **source_chapter**: 《十亿美金的教训》前言
- **failure_mode**: 创始人因个人英雄主义而自我迷失，无法升级为商业领袖。
- **mechanism**: 初期成功强化个人判断，后期组织、资本和用户现实被创始人意志覆盖。
- **bound_to**: [failure-forensics-map, talent-culture-operating-system]

## CE05 坏消息被愿景和融资掩盖

- **source_chapter**: 《创业维艰》《创业之殇》《大败局》
- **failure_mode**: 现金、产品、客户或组织问题已经恶化，但团队继续讲增长、融资和宏大愿景。
- **mechanism**: 创始人害怕士气崩塌，投资人和团队也愿意相信更好听的叙事。
- **bound_to**: [hard-things-wartime-leadership, failure-forensics-map]

## CE06 管理者忙于个人产出

- **source_chapter**: 《格鲁夫给经理人的第一课》
- **failure_mode**: 管理者把自己当最强执行者，忽略培训、信息流、决策机制和团队杠杆。
- **mechanism**: 创业早期个人英雄有效，规模扩大后反而成为瓶颈。
- **bound_to**: [manager-output-leverage]
