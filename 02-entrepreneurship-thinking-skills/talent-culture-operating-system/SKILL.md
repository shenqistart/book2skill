# 人才文化操作系统

## R — 原文 (Reading)

> “一定要找有共同的愿景、有能力、有责任心的人。”
>
> 来源：雷军《小米创业思考》第一章。

## I — 方法论骨架 (Interpretation)

文化不是墙上的词，而是一套让合适的人做出一致选择的操作系统。

这个 skill 把人才文化拆成四件事：选什么人，如何让他们理解使命，怎样给责任和信息，如何培养更多经营者。没有这些机制，创始人再强也会成为瓶颈。

它适合公司从十几人进入几十人、上百人，或者已经出现“价值观都懂但动作不一致”的阶段。

## A1 — 书中的应用 (Past Application)

### 案例 1：小米早期高密度组建合伙人团队

- **问题**：小米想做软件、硬件、互联网融合的复杂模式。
- **方法论的使用**：雷军用极高标准寻找共同愿景、有能力、有责任心且自驱的人。
- **结论**：复杂创业不能只靠管理盯人，必须先提高人才密度和信任基础。
- **结果**：早期团队支撑了小米的高强度探索。

### 案例 2：Google 强调创意精英环境

- **问题**：知识型公司如何保持创新和执行。
- **方法论的使用**：Google 以高人才密度、开放信息、使命和文化机制吸引并放大创意精英。
- **结论**：文化要服务人才自主判断，而不是让管理层控制一切。
- **结果**：组织能在复杂技术和产品环境中保持速度。

### 案例 3：柳井正强调店长和经营者培养

- **问题**：连锁零售如何避免总部一人说了算。
- **方法论的使用**：把店长视为公司主角，培养能独立判断的经营者。
- **结论**：一线责任人必须具备经营意识。
- **结果**：组织学习不只停留在总部。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 公司扩张后创始人无法再靠个人判断管理所有事。
2. 团队价值观说得一致，但真实决策冲突很多。
3. 招来的人能力强但不自驱，或自驱但方向不一致。
4. 一线负责人只执行命令，没有经营者意识。

### 语言信号

- “公司变大后文化变虚了。”
- “怎么定义我们要的人？”
- “团队都等老板，不像经营者。”
- “价值观怎么落到机制里？”

### 与相邻 skill 的区分

- 与 `manager-output-leverage` 的区别：后者优化管理动作；本 skill 设计人才和文化系统。
- 与 `failure-forensics-map` 的区别：本 skill 是事前组织设计；后者复盘文化和组织失灵。

## E — 可执行步骤 (Execution)

当 skill 被激活后，agent 应按以下步骤执行：

1. **定义关键人才画像**
   - 完成标准：写出能力、责任心、价值观、学习速度和自驱标准，并说明哪些特质一票否决。

2. **把使命翻译成行为**
   - 完成标准：把 3-5 条价值观分别转成可观察行为和反行为。

3. **设计信息和责任机制**
   - 完成标准：说明哪些信息默认透明，哪些决策下放，负责人承担什么结果。

4. **建立培养路径**
   - 完成标准：为新人、骨干、一线负责人和管理者分别定义成长动作。

5. **设置文化校验**
   - 完成标准：用招聘通过率、决策一致性、内部流动、复盘质量和一线自主性做反馈。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只是要写招聘 JD、价值观海报或团建活动。
- 具体员工违纪、绩效或劳动法律问题需要 HR/法律流程。
- 公司正处在裁员或生存危机，应先使用 `hard-things-wartime-leadership`。

### 作者在书中警告的失败模式

- 只讲文化，不改变招聘、授权、信息流和评价机制。
- 把人才密度误解为只招名校大厂背景。
- 创始人不愿下放责任，却抱怨团队没有经营者意识。

### 作者的盲点 / 时代局限

- Google、优衣库、小米的人才系统都依赖具体行业和阶段，不能机械复制。
- 高人才密度可能提高协作成本和文化排异，需要包容不同角色。

### 容易混淆的邻近方法论

- 企业文化宣言：本 skill 要求有机制和反馈。
- 绩效管理：绩效是文化系统的一部分，不是全部。
