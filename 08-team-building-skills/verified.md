# 通过三重验证的 skill 单元

每个单元按 book2skill Stage 1.5 进行三重验证：V1 至少两个独立语境支撑；V2 可迁移到新问题；V3 不是普通常识。

| Skill | V1: 支撑 | V2: 可迁移问题 | V3: 非常识性 |
|---|---|---|---|
| `decision-checklist-lattice` | 逆向思维、能力圈、多元模型、误判心理和检查清单互相支撑 | 适用于战略取舍、招聘、组织设计、风险评审 | 不是“多想想”，而是强制从失败路径和模型盲区审计 |
| `inside-out-trust-shift` | 主动性、影响圈、以终为始、要事优先、互赖协作构成链条 | 适用于冲突复盘、责任澄清、跨团队合作 | 不是“态度好”，而是把回应选择和影响范围变成行动边界 |
| `daily-highlight-focus-loop` | Highlight、Laser、Energize、Reflect 四步闭环 | 适用于个人和小团队从忙碌切换到重点交付 | 不是普通待办清单，而是用注意力和能量保护最重要事项 |
| `pyramid-communication-logic` | 结论先行、上下统摄、逻辑顺序、问答式展开和 SCQA 互相支撑 | 适用于战略 memo、OKR 汇报、复盘材料、组织变革说明 | 不是排版模板，而是读者问题驱动的论证结构 |
| `okr-quality-design` | 两本 OKR 书都强调目标少而关键、O/KR 区分、结果可验证、非绩效工具 | 适用于季度目标、团队重点、项目目标重写 | 不是“设目标”，而是检查目标是否能驱动对齐和结果 |
| `okr-alignment-review-cadence` | OKR 公开透明、纵横对齐、评分、季度复盘、CFR 构成运行节奏 | 适用于目标漂移、跨部门依赖、复盘低效 | 不是开更多会，而是把目标系统变成可追踪的管理节奏 |
| `freedom-responsibility-culture` | 奈飞把自由、责任、成年人假设、坦诚、纪律和少流程反复绑定 | 适用于减少低价值审批、提升团队自治和信息透明 | 不是“放手不管”，而是用上下文与责任替代流程控制 |
| `talent-density-role-fit` | 奈飞把高绩效团队、岗位未来要求、反馈、角色错配与团队建设绑定 | 适用于招聘、调岗、绩效问题和团队能力盘点 | 不是“淘汰低绩效”，而是先判断未来任务和角色匹配 |
