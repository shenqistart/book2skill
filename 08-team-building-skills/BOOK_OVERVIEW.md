# 08 团队建设 Book Overview

## 分类范围

`08_团队建设` 覆盖 7 本可用源文本，形成一条从个人判断到组织系统的知识链：

1. **决策质量**：《穷查理宝典》
2. **个人与互赖效能**：《高效能人士的七个习惯》《创造时间》
3. **结构化沟通**：《金字塔原理》
4. **目标管理系统**：《OKR：源于英特尔和谷歌的目标管理利器》《这就是OKR》
5. **文化与人才系统**：《奈飞文化手册》

一句话主旨：团队建设不是单点激励，而是把清醒决策、个人效能、结构化沟通、目标节奏、文化责任和人才适配连成一个可运行系统。

## 书之间的互补关系

- 《穷查理宝典》提供高风险判断的“先防错”底座，适合在目标、人才、文化方案确定前做反向审计。
- 《七个习惯》提供从个人主动性到互赖协作的行为底座，适合处理责任、信任和合作问题。
- 《创造时间》把个人效能压缩成每日可执行循环，适合把团队目标落到注意力和时间安排。
- 《金字塔原理》提供表达和讨论结构，适合把目标、文化、人才判断变成能被组织理解的材料。
- 两本 OKR 书共同构成目标系统：尼文/拉莫尔特偏设计与落地步骤，杜尔偏案例、组织动员和 CFR。
- 《奈飞文化手册》把目标之外的文化、人才、责任和信息透明补齐，避免 OKR 只剩指标系统。

## 核心矛盾

1. **目标多 vs 注意力少**：OKR 与《创造时间》共同要求少数重点。
2. **自由 vs 控制**：《奈飞文化手册》要求用上下文、责任和人才密度替代流程堆叠。
3. **坦诚 vs 伤害**：《七个习惯》和《奈飞文化手册》都强调信任与成年人假设，防止坦诚变成攻击。
4. **表达漂亮 vs 论证可靠**：《金字塔原理》和《穷查理宝典》共同要求结构与反向检查。
5. **个人努力 vs 系统设计**：个人效能必须接入目标、会议、协作和文化节奏。

## 产出的 8 个 skills

| 层级 | Skill | 来源 | 用途 |
|---|---|---|---|
| 决策底座 | `decision-checklist-lattice` | 《穷查理宝典》 | 审计重大团队决策的盲区和失败路径 |
| 信任协作 | `inside-out-trust-shift` | 《七个习惯》 | 把冲突、责任和信任问题转成影响圈行动 |
| 个人执行 | `daily-highlight-focus-loop` | 《创造时间》 | 把团队目标落成每日重点和注意力防护 |
| 组织表达 | `pyramid-communication-logic` | 《金字塔原理》 | 组织目标、复盘和变革沟通材料 |
| 目标设计 | `okr-quality-design` | 两本 OKR 书 | 设计高质量 Objective 与 Key Results |
| 目标运行 | `okr-alignment-review-cadence` | 两本 OKR 书 | 建立对齐、追踪、评分、复盘和 CFR 节奏 |
| 文化系统 | `freedom-responsibility-culture` | 《奈飞文化手册》 | 用自由、责任、坦诚和纪律减少低价值流程 |
| 人才系统 | `talent-density-role-fit` | 《奈飞文化手册》 | 判断角色适配、未来能力和团队人才密度 |

## 批判与边界

- OKR 不是万能管理系统，无法替代领导判断、战略选择和文化建设。
- Netflix 方法高度依赖信息透明、人才密度和成熟管理者，低信任组织不能只复制“自由”。
- 《七个习惯》和《创造时间》偏个人行为改变，遇到组织结构性约束时必须上升到 OKR、沟通和文化层。
- 《金字塔原理》能提升表达清晰度，但不能弥补事实不足或战略错误。
- 《穷查理宝典》的模型方法适合审计判断，不应被 agent 用来做投资或人生裁判。
