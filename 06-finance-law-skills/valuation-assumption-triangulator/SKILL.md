# 估值假设三角校验

## R — 原文 (Reading)

> “估值不是数字游戏，而是关于企业的故事。”
>
> “现金流、增长和风险必须彼此一致。”
>
> 来源：阿斯瓦斯·达摩达兰《应用公司理财》估值相关章节。

## I — 方法论骨架 (Interpretation)

估值模型最危险的地方，不是公式复杂，而是假设互相打架。

这个 skill 把估值从“给一个数”改成“审计一个故事”：增长是否需要再投资支撑，风险是否匹配折现率，现金流口径是否和资本结构一致，倍数比较是否可比，融资条款是否改变价值归属。

它适合让 agent 发现估值里的不一致，而不是预测市场价格。

## A1 — 书中的应用 (Past Application)

### 案例 1：DCF 估值

- **问题**：企业价值取决于未来现金流。
- **方法论的使用**：预测自由现金流，并用与风险匹配的折现率贴现。
- **结论**：DCF 的可信度来自假设一致性。
- **结果**：估值从数字变成可审计的经营故事。

### 案例 2：相对估值

- **问题**：用户想用同业倍数快速估值。
- **方法论的使用**：检查可比公司、增长、利润率、风险和会计口径。
- **结论**：倍数不是捷径，背后仍是现金流、增长和风险。
- **结果**：避免用不可比倍数包装高估值。

### 案例 3：创业融资估值

- **问题**：融资 BP 用高速增长支撑估值。
- **方法论的使用**：检查增长是否需要销售、研发、营运资本和组织投入。
- **结论**：高增长必须有再投资和风险假设支撑。
- **结果**：发现“增长故事”和财务模型的断裂。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 审查 DCF、估值倍数、融资 BP 或并购报价。
2. 估值故事听起来好，但现金流、增长、风险不一致。
3. 用户想比较多个估值口径。
4. 融资结构、优先权或稀释可能影响价值归属。

### 语言信号

- “这个估值靠谱吗？”
- “DCF 假设有没有问题？”
- “这个倍数能不能用？”
- “融资 BP 的估值故事是否自洽？”

### 与相邻 skill 的区分

- 与 `capital-budgeting-npv-discipline` 的区别：NPV 判断项目，本 skill 审计企业或交易估值。
- 与 `risk-return-hurdle-rate` 的区别：后者定折现率，本 skill 检查所有估值假设一致性。
- 与 `capital-structure-apv-check` 的区别：融资副效应复杂时需要联用。

## E — 可执行步骤 (Execution)

1. **锁定估值对象**
   - 完成标准：明确估的是企业、股权、项目、业务线、并购标的还是融资轮次。

2. **拆现金流口径**
   - 完成标准：区分 FCFF、FCFE、会计利润、EBITDA、GMV 或收入，指出口径风险。

3. **检查增长和再投资**
   - 完成标准：说明增长需要什么资本开支、营运资本、销售投入或组织能力。

4. **匹配风险和折现率**
   - 完成标准：检查风险来源、资本成本和现金流币种/通胀/名义口径是否一致。

5. **交叉相对估值**
   - 完成标准：比较可比公司、倍数口径、增长、利润率、风险和控制权差异。

6. **输出估值审计结论**
   - 完成标准：列出可信假设、脆弱假设、缺失数据和不能下结论的部分。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要求股票目标价、买卖建议或收益保证。
- 现金流、增长和风险输入完全缺失。
- 合同条款、税务或法律效力需要专业意见。

### 作者在书中警告的失败模式

- 用复杂模型掩盖假设不一致。
- 用不可比公司的倍数支持想要的价格。
- 高增长假设没有再投资支撑。

### 作者的盲点 / 时代局限

- 市场情绪、流动性、监管和融资环境会改变交易价格。
- 创业公司早期数据稀少，估值更像假设审计而非精确计算。
