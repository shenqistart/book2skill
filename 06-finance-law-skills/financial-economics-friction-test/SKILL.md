# 金融经济学模型前提检查

## R — 原文 (Reading)

> “金融经济学有三条主线：均衡资产定价、无套利资产定价和金融摩擦。”
>
> “模型有用，但前提失效时会误导判断。”
>
> 来源：徐高《金融经济学二十五讲》。

## I — 方法论骨架 (Interpretation)

当金融模型解释不了现实，不要只说“市场不理性”。先问你用了哪类模型，它需要什么前提。

均衡模型关心风险和预期收益如何匹配；无套利模型关心能否复制现金流；金融摩擦模型关心交易限制、信息不对称、杠杆约束、流动性和制度边界。很多“套利机会”没有消失，是因为摩擦让套利不可执行。

## A1 — 书中的应用 (Past Application)

### 案例 1：CAPM

- **问题**：资产收益无法被简单 beta 解释。
- **方法论的使用**：检查系统性风险、市场组合、可分散风险和估计窗口。
- **结论**：CAPM 需要严格前提。
- **结果**：避免把所有风险都塞进一个 beta。

### 案例 2：无套利定价

- **问题**：衍生品或债券价格看似存在套利。
- **方法论的使用**：检查复制组合、交易成本、做空限制和流动性。
- **结论**：无套利需要可复制、可交易、低摩擦。
- **结果**：区分理论套利和可执行套利。

### 案例 3：金融摩擦

- **问题**：市场价格偏离基本价值却长期存在。
- **方法论的使用**：分析信息、杠杆、流动性、监管和机构约束。
- **结论**：摩擦可能让价格偏离持续。
- **结果**：模型解释从完美市场转向现实约束。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 金融模型结果与现实价格明显冲突。
2. 用户问套利机会为什么没有消失。
3. CAPM、有效市场或风险中性定价被机械套用。
4. 估值或资本成本争议来自模型前提不同。

### 语言信号

- “CAPM 为什么解释不了？”
- “这是不是套利机会？”
- “有效市场假设失效了吗？”
- “模型前提是不是不成立？”

### 与相邻 skill 的区分

- 与 `risk-return-hurdle-rate` 的区别：后者设门槛，本 skill 审查模型前提。
- 与 `valuation-assumption-triangulator` 的区别：估值审计看企业假设，本 skill 看金融理论假设。
- 与 `capital-structure-apv-check` 的区别：融资结构问题可用摩擦视角解释契约和违约成本。

## E — 可执行步骤 (Execution)

1. **识别模型类型**
   - 完成标准：判断问题属于均衡定价、无套利复制还是金融摩擦。

2. **列出模型前提**
   - 完成标准：写出所需的市场组合、可交易性、复制、低交易成本、信息对称等前提。

3. **检查前提失效点**
   - 完成标准：标出流动性、做空、杠杆、税费、监管、信息和行为约束。

4. **区分理论与执行**
   - 完成标准：说明理论价格、可交易价格和可执行策略的差异。

5. **回连资本决策**
   - 完成标准：指出对折现率、估值、融资或风险管理的影响。

6. **输出模型使用边界**
   - 完成标准：说明模型可用、需修正、不可用或信息不足。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要求证券买卖建议、套利交易指令或收益承诺。
- 缺少价格、现金流、可交易性或市场结构信息。
- 问题是公司项目价值，应先走 NPV 或估值 skill。

### 作者在书中警告的失败模式

- 忘记模型前提。
- 把理论套利当成无风险赚钱机会。
- 用单一模型解释所有金融现象。

### 作者的盲点 / 时代局限

- 金融市场制度、交易规则和数据环境高度时变。
- 行为金融、监管冲击和极端流动性危机可能超出标准模型。
