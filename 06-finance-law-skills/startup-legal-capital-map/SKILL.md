# 创业法律资本地图

## R — 原文 (Reading)

> “创业者应当重视公司设立、股权结构和公司治理。”
>
> 来源：杨春宝、王成兵《创业投资法律手册》。

## I — 方法论骨架 (Interpretation)

创业资本动作必须被法律文件承接。

融资估值、股权分配、期权激励、技术出资、知识产权、章程条款、投资协议、劳动关系和退出安排，都会改变经济结果。这个 skill 不做法律判断，而是帮助 agent 知道问题落在哪些文件、权利、义务和专业审查边界上。
它尤其适合把“股权/融资看起来合理”的方案，翻译成“哪些法律事实还没确认”。

## A1 — 书中的应用 (Past Application)

### 案例 1：公司设立和出资

- **问题**：创始人出资、认缴、实缴和资产投入不清。
- **方法论的使用**：检查公司设立文件、出资义务和股东责任。
- **结论**：资本安排不能只靠口头约定。
- **结果**：需要把权利义务写进文件并核验登记。

### 案例 2：股权和控制权

- **问题**：股权比例看似公平，但治理和表决机制缺失。
- **方法论的使用**：检查章程、股东会、董事会、表决权、转让限制和退出机制。
- **结论**：股权比例不等于控制权安全。
- **结果**：融资前必须审查治理结构。

### 案例 3：知识产权和劳动关系

- **问题**：核心代码、商标或发明来自员工、外包或合作方。
- **方法论的使用**：检查知识产权归属、保密、竞业和劳动合同。
- **结论**：知识产权瑕疵会影响融资和并购。
- **结果**：法律地图提前暴露交易障碍。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 创业者准备设立公司、分股或做期权激励。
2. 融资前需要检查章程、股权、知识产权和合同风险。
3. 投资协议、股权转让、对赌或控制权条款影响财务结果。
4. 财务方案需要在中国法律环境下落地。

### 语言信号

- “这些股权安排有什么法律坑？”
- “融资前要检查哪些文件？”
- “技术出资和知识产权怎么确认风险？”
- “投资协议里的条款会影响控制权吗？”

### 与相邻 skill 的区分

- 与 `finance-question-router` 的区别：路由器识别法律入口；本 skill 展开法律资本地图。
- 与 `valuation-assumption-triangulator` 的区别：估值看经济假设；本 skill 看法律文件和权利义务。
- 与 `cash-distribution-policy` 的区别：现金政策看资本配置；本 skill 检查分配的法律和契约边界。

## E — 可执行步骤 (Execution)

1. **界定资本动作**
   - 完成标准：说明是设立、出资、分股、融资、激励、转让、并购还是退出。

2. **映射关键文件**
   - 完成标准：列出章程、股东协议、投资协议、劳动合同、IP 文件、债务合同和登记文件。

3. **检查权利义务**
   - 完成标准：标记出资义务、表决权、转让限制、优先权、对赌、回购和退出安排。

4. **检查资产和知识产权**
   - 完成标准：说明核心资产、代码、商标、专利、数据和外包成果的归属风险。

5. **标记专业复核点**
   - 完成标准：列出必须交由律师、税务或会计专业人士核验的问题。

6. **输出风险地图**
   - 完成标准：按高/中/低风险列出风险、影响和下一步取证材料，不给法律意见。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要求直接起草合同、出具法律意见或判断条款有效性。
- 用户要求规避监管、隐瞒股权或设计违法安排。
- 涉及实时法律、司法解释、税务和监管口径，必须查证并由专业人士复核。

### 作者在书中警告的失败模式

- 口头分股。
- 忽略章程和股东协议。
- 知识产权归属不清却进行融资。

### 作者的盲点 / 时代局限

- 法律规则会变化，本 skill 只能做问题地图。
- 不同地区、行业和交易结构会有额外监管要求。
