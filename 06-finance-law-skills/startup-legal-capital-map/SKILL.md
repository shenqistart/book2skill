# 创业法律资本地图

## R — 原文 (Reading)

> “创业企业的资本动作，最终都要落到法律文件和治理结构中。”
>
> “股权、章程、合同和知识产权，是创业法律风险的高发区。”
>
> 来源：杨春宝、王成兵《创业投资法律手册》。

## I — 方法论骨架 (Interpretation)

财务方案只有在法律上可执行，才是真方案。

创业公司常把融资、股权、期权、技术出资、代持、章程和投资协议当作商业谈判问题，但它们最终会进入公司法、合同、知识产权、劳动和税务边界。本 skill 不给法律结论，而是让 agent 知道哪里必须停下来做专业复核。

## A1 — 书中的应用 (Past Application)

### 案例 1：公司设立与章程

- **问题**：创始团队只口头约定权利义务。
- **方法论的使用**：检查公司形式、出资、章程、表决权和退出机制。
- **结论**：治理规则需要文件化。
- **结果**：降低后续控制权和分配争议。

### 案例 2：知识产权

- **问题**：核心技术或作品归属不清。
- **方法论的使用**：检查职务成果、转让协议、许可和保密义务。
- **结论**：IP 归属会直接影响融资和估值。
- **结果**：融资前必须清理权属。

### 案例 3：投资协议

- **问题**：投资条款涉及优先权、对赌和回购。
- **方法论的使用**：将条款映射到治理、现金流、控制权和法律效力风险。
- **结论**：条款经济效果和法律边界要一起审查。
- **结果**：避免只看估值忽略权利负担。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 创业公司设立、股权分配、代持、期权或章程问题。
2. 融资、投资协议、清算优先、对赌、回购或控制权条款。
3. 知识产权、技术出资、员工成果和保密问题。
4. 财务方案需要落到合同和治理文件。

### 语言信号

- “章程要注意什么？”
- “投资协议有哪些法律坑？”
- “技术出资/IP 归属怎么处理？”
- “创始人股权和员工期权怎样落地？”

### 与相邻 skill 的区分

- 与 `capital-structure-apv-check` 的区别：后者看融资经济效应，本 skill 看条款和治理边界。

## E — 可执行步骤 (Execution)

1. **定位法律事项**
   - 完成标准：判断问题属于设立、股权、治理、合同、IP、劳动、融资或并购。

2. **映射关键文件**
   - 完成标准：列出章程、股东协议、投资协议、劳动/IP 协议、合同或审批文件。

3. **标记权利义务**
   - 完成标准：提取表决权、分红权、优先权、回购、限制转让、竞业和保密等权利。

4. **连接资本影响**
   - 完成标准：说明条款如何影响现金流、控制权、稀释、估值或退出。

5. **列专业复核清单**
   - 完成标准：明确哪些点需要律师、税务、会计或工商登记复核。

6. **输出边界建议**
   - 完成标准：给出风险地图、缺失资料和不能由 agent 下结论的事项。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要求判断合同条款最终效力、诉讼胜率或规避法律。
- 用户要求起草可直接签署的法律文件。
- 涉及不同司法辖区但缺少适用法律和事实。

### 作者在书中警告的失败模式

- 口头约定股权和职责。
- 融资只看估值，不看投资人权利。
- IP、劳动和合同问题融资前才暴露。

### 作者的盲点 / 时代局限

- 法律法规、司法解释和监管口径会变化。
- 本 skill 只能做问题地图，不能替代执业律师。
