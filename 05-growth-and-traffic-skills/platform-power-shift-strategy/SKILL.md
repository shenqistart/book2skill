# 平台权力迁移策略

## R — 原文 (Reading)

> “这些数据使得特德·萨兰多斯相信，《纸牌屋》的创意完全行得通。”
>
> “数字分发改变了内容行业的权力结构。”
>
> 来源：Michael D. Smith、Rahul Telang《流媒体时代》。

## I — 方法论骨架 (Interpretation)

平台不是渠道表格中的一个选项，而是会改变谁拥有用户、数据、定价权和分发权的制度环境。

《流媒体时代》通过 Netflix、iTunes、亚马逊、NBC 和盗版案例说明：数字平台用数据、推荐、捆绑、入口、用户关系和版权规则重写行业权力。增长策略必须判断自己是在借平台势能，还是把核心用户资产交给平台。

## A1 — 书中的应用 (Past Application)

### 案例 1：Netflix

- **问题**：平台如何决定自制内容。
- **方法论的使用**：利用用户数据和观看行为判断内容机会。
- **结论**：数据让平台从分发者转为内容决策者。
- **结果**：平台掌握更多产业权力。

### 案例 2：iTunes

- **问题**：数字分发如何改变音乐购买。
- **方法论的使用**：拆解单曲销售、定价和版权控制。
- **结论**：数字渠道改变内容打包和收入分配。
- **结果**：旧的专辑和渠道模式被重写。

### 案例 3：盗版与流媒体

- **问题**：盗版和便利性如何影响付费意愿。
- **方法论的使用**：比较价格、便利、可得性和用户体验。
- **结论**：平台竞争不只是内容数量，也是用户体验和可得性。
- **结果**：合法平台通过便利性重夺需求。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 平台规则、抽佣、算法或推荐机制改变。
2. 内容方、品牌或商家考虑是否依赖平台。
3. 用户想判断自有渠道、私域、会员或平台分发的取舍。
4. 直播、短视频、流媒体或内容增长受平台权力影响。

### 语言信号

- “平台规则变了怎么办？”
- “要不要做自有渠道？”
- “内容方会不会被平台卡住？”
- “算法推荐改变后增长怎么调整？”

### 与相邻 skill 的区分

- 与 `bullseye-channel-selection` 的区别：本 skill 先看平台权力和依赖边界，渠道选择再决定测试。
- 与 `live-commerce-fit-map` 的区别：直播是平台场景之一，本 skill 看更高层的权力结构。
- 与 `relationship-flywheel-private-domain` 的区别：私域是降低平台依赖的一种关系资产。

## E — 可执行步骤 (Execution)

1. **识别平台角色**
   - 完成标准：判断平台是流量入口、交易场、内容库、数据拥有者、支付方还是规则制定者。

2. **拆权力来源**
   - 完成标准：列出平台掌握的数据、推荐、用户关系、定价、版权、抽佣和封禁权。

3. **评估依赖程度**
   - 完成标准：衡量收入、获客、内容曝光、用户数据和品牌认知对平台的依赖。

4. **寻找可保留资产**
   - 完成标准：设计自有会员、邮件、私域、品牌搜索、内容 IP 或多平台资产。

5. **判断策略选项**
   - 完成标准：给出继续深耕、分散平台、自建渠道、谈判合作或退出的条件。

6. **设置监控指标**
   - 完成标准：监控平台抽佣、自然分发占比、用户可迁移性、数据可得性和规则风险。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只是要短期内容选题、标题或单条视频优化。
- 需要最新平台政策、费率、算法细则，必须实时查证。
- 涉及版权、合规或合同争议，需要专业法律意见。

### 作者在书中警告的失败模式

- 把平台流量误认为自有用户资产。
- 忽视数据和推荐权带来的产业权力迁移。
- 只看短期收入，不看长期谈判地位。

### 作者的盲点 / 时代局限

- 平台生态变化极快，书中案例不能替代当前平台情报。
- 不同行业的内容版权、监管和用户迁移成本差异很大。
