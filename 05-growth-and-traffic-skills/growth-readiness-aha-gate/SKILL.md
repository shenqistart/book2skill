# 增长门禁与啊哈时刻识别

## R — 原文 (Reading)

> “如果不能使用这个产品，你会有多失望？”
>
> “40% 的用户回答非常失望。”
>
> 来源：肖恩·埃利斯、摩根·布朗《增长黑客》。

## I — 方法论骨架 (Interpretation)

增长不是在任何产品上都能启动的外部机器。

《增长黑客》把增长前置为门禁：产品必须让一部分用户感到不可或缺，并且能在行为数据中看到用户抵达价值的关键时刻。
如果没有啊哈时刻、留存和热爱用户，渠道放大只会放大流失和浪费。

这个 skill 用来决定：现在该增长、该修产品，还是该重新定位目标用户。

## A1 — 书中的应用 (Past Application)

### 案例 1：不可或缺性调查

- **问题**：如何判断产品是否值得进入增长阶段。
- **方法论的使用**：询问用户如果不能继续使用产品会有多失望。
- **结论**：足够比例的“非常失望”用户，说明产品具备增长基础。
- **结果**：增长团队不是凭直觉开动，而是先检查用户热爱度。

### 案例 2：Twitter 的关注数

- **问题**：用户注册后为什么留不下来。
- **方法论的使用**：分析行为数据，发现关注足够数量账号后更容易留存。
- **结论**：增长要围绕用户真正感知价值的行为设计。
- **结果**：激活优化从注册表单转向引导用户完成关键行为。

### 案例 3：Slack 的团队消息量

- **问题**：协作工具何时真正被团队采用。
- **方法论的使用**：把团队发送足够消息视为进入协作价值的信号。
- **结论**：啊哈时刻常是一个行为阈值，而不是一句卖点。
- **结果**：增长重点转为让用户更快抵达价值。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 产品刚上线，团队想立刻投放或裂变。
2. 注册、下载或试用不少，但激活和留存不稳。
3. 团队说不清用户第一次真正获得价值的时刻。
4. 用户群体混杂，需要找出最热爱产品的细分人群。

### 语言信号

- “我们能不能开始大规模增长？”
- “产品还没留存，要不要先买量？”
- “啊哈时刻怎么找？”
- “怎么判断 PMF 足不足够？”

### 与相邻 skill 的区分

- 与 `repeat-customer-experience-loop` 的区别：本 skill 判断产品价值是否可放大；前者检查体验能否沉淀复购。
- 与 `north-star-growth-model` 的区别：本 skill 是增长门禁；北极星在门禁通过后统一增长模型。
- 与 `bullseye-channel-selection` 的区别：渠道选择应该发生在增长门禁通过之后。

## E — 可执行步骤 (Execution)

1. **定义候选啊哈时刻**
   - 完成标准：列出 2-4 个用户可能首次感到价值的行为或结果。

2. **做不可或缺性调查**
   - 完成标准：询问核心用户“如果不能继续使用会多失望”，并记录非常失望比例和原因。

3. **检验行为阈值**
   - 完成标准：用激活、留存或复购数据找出与长期使用相关的关键行为阈值。

4. **分离热爱用户**
   - 完成标准：描述最热爱用户的人群、使用场景、需求强度和获益方式。

5. **判断增长状态**
   - 完成标准：给出“可增长 / 先修产品 / 重定位用户 / 继续探索”的门禁结论。

6. **设计下一步动作**
   - 完成标准：如果通过，转北极星和实验系统；如果未通过，列出产品价值或定位修复动作。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户已经有稳定留存，只是要比较渠道或优化出价。
- 用户要求实时平台规则、广告素材建议或短期投放复盘。
- 产品是强合规或一次性项目，无法用常规留存衡量，应改用对应业务指标。

### 作者在书中警告的失败模式

- 没有产品价值就急着做增长。
- 用注册量、下载量或曝光量替代不可或缺性。
- 没有找到热爱用户，就把所有人当成目标用户。

### 作者的盲点 / 时代局限

- 40% 门槛来自实践经验，不是所有品类的绝对标准。
- B2B、企业采购和低频工具可能需要更长观察窗口。
- 书中案例多来自软件和互联网产品，实体业务要转换指标。
