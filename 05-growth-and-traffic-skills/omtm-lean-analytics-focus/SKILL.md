# OMTM 精益指标聚焦

## R — 原文 (Reading)

> “找到现阶段最重要的那个指标。”
>
> “虚荣指标会让你感觉良好。”
>
> 来源：阿利斯泰尔·克罗尔、本杰明·尤科维奇《精益数据分析》。

## I — 方法论骨架 (Interpretation)

数据的价值不在于多，而在于迫使团队做出正确行动。

《精益数据分析》把创业公司数据问题拆成阶段、商业模式、底线和 OMTM。
一个阶段只需要一个最重要指标，因为团队的注意力、实验和学习都很稀缺。
如果指标不能改变下一步行动，它就只是报表装饰。

这个 skill 用来从一堆数据中选出当前最该推动的指标，并定义阈值和行动。

## A1 — 书中的应用 (Past Application)

### 案例 1：虚荣指标

- **问题**：访问量、注册量、下载量增长，看起来很好但不指导行动。
- **方法论的使用**：区分虚荣指标和可行动指标。
- **结论**：指标必须能指导下一步改变。
- **结果**：团队停止用好看的数字自我安慰。

### 案例 2：Airbnb 摄影质量

- **问题**：供给端房源转化受图片质量影响。
- **方法论的使用**：选择阶段性关键指标，把实验集中在摄影改进。
- **结论**：OMTM 应该对应当前最大瓶颈。
- **结果**：数据焦点推动具体运营动作。

### 案例 3：六种商业模式

- **问题**：不同业务不能用同一套指标。
- **方法论的使用**：按电商、SaaS、媒体、双边市场等模型选择指标。
- **结论**：指标要匹配商业模式和创业阶段。
- **结果**：数据分析从通用报表转为业务模型判断。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 数据很多，但团队不知道该看哪个。
2. 指标增长漂亮，却无法解释业务是否变好。
3. 团队需要为当前阶段选择一个最重要指标。
4. 用户希望用数据决定下一轮增长实验。

### 语言信号

- “我们现在最该看哪个指标？”
- “这些数据是不是虚荣指标？”
- “OMTM 怎么定？”
- “数据会开了很多但没有行动。”

### 与相邻 skill 的区分

- 与 `north-star-growth-model` 的区别：北极星是长期方向；OMTM 是当前阶段焦点。
- 与 `growth-experiment-operating-system` 的区别：OMTM 选择要优化什么；实验系统决定怎么验证。
- 与 `bullseye-channel-selection` 的区别：渠道测试也需要 OMTM 来定义成功标准。

## E — 可执行步骤 (Execution)

1. **定位阶段和模型**
   - 完成标准：说明业务处于问题/方案匹配、MVP、留存、规模化或变现阶段，以及所属商业模式。

2. **列出候选指标**
   - 完成标准：列出当前团队正在看的指标，并标注获取、激活、留存、收入或推荐属性。

3. **剔除虚荣指标**
   - 完成标准：排除不能触发行动、不能分群、不能解释质量的漂亮数字。

4. **选择 OMTM**
   - 完成标准：选出当前唯一关键指标，并说明它为什么是最大瓶颈。

5. **定义阈值和分群**
   - 完成标准：给出目标阈值、时间窗口、同期群或关键分群，避免平均数误导。

6. **绑定下一步行动**
   - 完成标准：写出指标升降分别要采取的产品、运营、渠道或定价动作。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只是要整理展示报表，而没有决策问题。
- 用户需要实时广告平台数据、财务审计或证券预测。
- 问题已经明确是组织协作失败，应转增长实验操作系统。

### 作者在书中警告的失败模式

- 用虚荣指标掩盖真实瓶颈。
- 同时追太多指标，团队失去焦点。
- 不按阶段和商业模式选择指标。

### 作者的盲点 / 时代局限

- OMTM 过度简化时可能忽略护栏指标。
- 数据质量差时，选择指标前要先修埋点和口径。
- 高监管行业不能只按增长指标决策。
