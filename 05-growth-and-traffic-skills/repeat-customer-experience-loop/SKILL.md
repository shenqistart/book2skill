# 回头客体验循环

## R — 原文 (Reading)

> “把本应投到广告上的资金运用到提高客户服务和顾客体验上。”
>
> “千万不要外包你的核心职能部门。”
>
> 来源：谢家华《回头客战略》。

## I — 方法论骨架 (Interpretation)

增长不是只把更多陌生人买进来，而是让一次购买变成复购、推荐和品牌记忆。

《回头客战略》的关键不是“客服态度好”，而是把客户体验当作获客系统的一部分：广告预算、物流承诺、服务权限、员工文化和品牌故事都服务于同一个循环。
当顾客愿意回来并主动推荐时，企业就减少了对外部流量的依赖。

这个 skill 用来判断一个增长问题是否应该先修复体验资产，而不是加码投放。

## A1 — 书中的应用 (Past Application)

### 案例 1：美捷步减少广告依赖

- **问题**：电商获客依赖广告，长期成本会上升。
- **方法论的使用**：把预算转向客服、物流和惊喜体验，让顾客主动复购和传播。
- **结论**：体验可以成为长期获客资产。
- **结果**：品牌记忆由交易价格转为服务体验。

### 案例 2：把客服留在核心职能内

- **问题**：客服常被当作成本中心外包。
- **方法论的使用**：美捷步把客服视为品牌接触点，保留权限和训练。
- **结论**：一线服务如果脱离公司文化，体验循环会断裂。
- **结果**：客服不只是解决问题，而是在制造回头客。

### 案例 3：文化书和员工训练

- **问题**：服务标准不能只靠流程文件维持。
- **方法论的使用**：用文化、招聘、培训和授权让员工理解品牌承诺。
- **结论**：体验的一致性来自组织系统。
- **结果**：口碑不是活动结果，而是日常运营结果。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户说流量越来越贵，希望降低获客压力。
2. 用户有新客但复购低，想找长期增长杠杆。
3. 用户把客服、物流、售后和品牌分开管理，体验不一致。
4. 用户想做私域或会员，但老客关系基础薄弱。

### 语言信号

- “怎么降低流量成本？”
- “为什么用户买一次就走？”
- “客服、售后、物流要不要算增长？”
- “有没有比继续投广告更长期的办法？”

### 与相邻 skill 的区分

- 与 `growth-readiness-aha-gate` 的区别：本 skill 看体验能否沉淀复购；后者判断产品是否具备增长门禁。
- 与 `bullseye-channel-selection` 的区别：本 skill 先降低对外部渠道的依赖；后者选择获客渠道。
- 与 `relationship-flywheel-private-domain` 的区别：本 skill 从客户体验出发；后者从私域关系和超级用户出发。

## E — 可执行步骤 (Execution)

1. **拆分增长成本**
   - 完成标准：把当前增长投入分成新客投放、转化补贴、客服售后、物流体验、会员/私域和品牌内容。

2. **识别体验断点**
   - 完成标准：列出从下单、履约、售后、复购到推荐中最容易让用户流失或失望的 3 个断点。

3. **判断核心职能**
   - 完成标准：说明哪些触点直接影响品牌承诺，不能简单外包或只按成本压缩。

4. **把预算迁移为体验资产**
   - 完成标准：提出至少 2 个可从短期投放或补贴中迁移预算的体验改进项。

5. **设计复购和推荐触发**
   - 完成标准：为老客设计复购理由、推荐理由和被感谢的体验，而不是只给优惠券。

6. **建立循环指标**
   - 完成标准：用复购率、推荐率、客服满意度、解决时长、老客 GMV 占比和投诉复发率验证循环。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只要求广告平台出价、素材点击率或投放账户诊断。
- 产品价值还没有基本验证，老客体验无法弥补核心需求不存在。
- 业务是一次性、极低频且无推荐价值的交易。

### 作者在书中警告的失败模式

- 把客服当作可随意外包的成本中心。
- 用广告替代品牌承诺和服务体验。
- 口号上重视客户，组织权限和预算却不支持一线体验。

### 作者的盲点 / 时代局限

- 美捷步案例处在美国电商早期阶段，物流和平台竞争环境与今天不同。
- 强体验策略需要现金流和组织耐心，不适合所有低毛利业务直接照搬。
- 书中对实时平台流量和算法分发讨论较少。
