# 直播电商适配地图

## R — 原文 (Reading)

> “直播电商并没有改变人、货、场匹配的零售基本逻辑。”
>
> “信任是直播电商成交的关键资产。”
>
> 来源：《直播电商的逻辑》。

## I — 方法论骨架 (Interpretation)

直播电商不是“找个主播卖货”，而是人、货、场重新组合后的零售系统。

适合直播的品类通常有可展示性、可解释性、信任门槛、价格/福利感和供应链履约能力。主播不是流量容器，而是信任中介；平台不是中立货架，而会通过算法、规则和佣金改变利润分配。

## A1 — 书中的应用 (Past Application)

### 案例 1：主播信任

- **问题**：品牌以为只要流量大就能成交。
- **方法论的使用**：检查主播人设、专业度、用户信任和品类匹配。
- **结论**：不匹配的主播会消耗品牌信任。
- **结果**：选择主播应看信任适配，而非只看粉丝量。

### 案例 2：供应链履约

- **问题**：直播爆单后体验崩溃。
- **方法论的使用**：检查库存、发货、售后、退换货和价格体系。
- **结论**：直播会放大供应链弱点。
- **结果**：未准备好履约的品牌不应盲目放量。

### 案例 3：平台场域

- **问题**：同一商品在不同平台表现差异大。
- **方法论的使用**：比较平台用户、推荐机制、内容形态和转化链路。
- **结论**：平台决定“场”的规则。
- **结果**：直播策略必须按平台场域重写。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 评估品类是否适合直播电商。
2. 在达人直播、店播、自播、视频号或平台直播之间选择。
3. 直播 GMV 高但复购、利润或售后很差。
4. 用户只看主播流量，忽略供应链和信任。

### 语言信号

- “这个品类适合直播吗？”
- “该做达人直播还是店播？”
- “直播卖得多但亏钱怎么办？”
- “主播粉丝多就够了吗？”

### 与相邻 skill 的区分

- 与 `platform-power-shift-strategy` 的区别：本 skill 评估直播零售适配，后者看平台权力结构。
- 与 `repeat-customer-experience-loop` 的区别：直播成交后仍需复购和体验闭环。
- 与 `small-group-community-leverage` 的区别：直播社群只是场景之一，需要人货场检查。

## E — 可执行步骤 (Execution)

1. **检查货**
   - 完成标准：评估品类展示性、解释成本、毛利、价格体系、库存和售后。

2. **检查人**
   - 完成标准：评估主播/店员的信任、专业、用户匹配和品牌风险。

3. **检查场**
   - 完成标准：比较平台用户、内容形态、算法、佣金、支付和售后规则。

4. **检查履约**
   - 完成标准：确认供应链、客服、退换货和爆单承压能力。

5. **选择直播模式**
   - 完成标准：给出达人直播、店播、自播、视频号或暂缓的建议。

6. **定义复盘指标**
   - 完成标准：同时看 GMV、毛利、退款率、复购、客诉、粉丝资产和平台依赖。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只需要实时平台投放价格、坑位费或主播报价。
- 品类涉及医疗、金融、功效宣称等合规高风险内容。
- 供应链和售后完全无法承接短期爆量。

### 作者在书中警告的失败模式

- 只看主播流量，不看信任和品类适配。
- 只看 GMV，不看利润和售后。
- 把平台红利误认为品牌能力。

### 作者的盲点 / 时代局限

- 直播平台规则、监管、流量成本和用户疲劳变化很快。
- 不同国家/地区的直播电商成熟度差异很大。
