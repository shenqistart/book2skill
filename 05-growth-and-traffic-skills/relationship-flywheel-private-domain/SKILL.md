# 私域关系飞轮

## R — 原文 (Reading)

> “私域就是那些你直接拥有、可重复、低成本甚至免费触达的用户。”
>
> “关系飞轮的核心是关系，而不是工具。”
>
> 来源：车军《关系飞轮》。

## I — 方法论骨架 (Interpretation)

私域不是把用户拉进群，也不是多一个触达通道。私域的价值来自关系密度：用户愿意再次听你说话、相信你的推荐、参与反馈，并把关系介绍给相似的人。

本 skill 把私域设计分成三件事：用户为什么愿意留下，关系如何从弱触达变成信任，超级用户如何带来复购、内容、反馈和转介绍。

## A1 — 书中的应用 (Past Application)

### 案例 1：超级用户

- **问题**：普通用户很多，但复购和转介绍不足。
- **方法论的使用**：识别高信任、高贡献、高复购用户。
- **结论**：增长飞轮应围绕超级用户，而非平均用户。
- **结果**：关系资产开始替代一次性流量。

### 案例 2：亲密关系模拟

- **问题**：品牌触达像群发广告。
- **方法论的使用**：用可信、可亲、可爱三个维度设计互动。
- **结论**：关系不是频率，而是被用户感知到的价值和尊重。
- **结果**：触达从打扰转为服务。

### 案例 3：私域飞轮

- **问题**：私域转化越来越差。
- **方法论的使用**：检查内容、权益、反馈、复购和转介绍是否循环。
- **结论**：没有循环的私域只是沉默名单。
- **结果**：用关系动作重建飞轮。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 私域群、会员、企微或社群转化下降。
2. 用户希望提升复购、转介绍和超级用户贡献。
3. 用户把私域理解成“免费流量池”，需要纠偏。
4. 用户想设计长期关系飞轮，而不是短期促销。

### 语言信号

- “私域怎么做？”
- “超级用户怎么运营？”
- “复购和转介绍怎么提高？”
- “群里没人互动怎么办？”

### 与相邻 skill 的区分

- 与 `repeat-customer-experience-loop` 的区别：后者先检查体验和复购底座，本 skill 设计关系资产。
- 与 `small-group-community-leverage` 的区别：本 skill 是关系飞轮，小群 skill 是社群结构杠杆。
- 与 `bullseye-channel-selection` 的区别：私域是候选渠道之一，但不是所有品类默认核心渠道。

## E — 可执行步骤 (Execution)

1. **定义关系对象**
   - 完成标准：区分普通用户、活跃用户、复购用户、超级用户和潜在推荐者。

2. **检查留存理由**
   - 完成标准：说明用户留在私域中得到什么持续价值，而不是只收广告。

3. **设计关系动作**
   - 完成标准：给出可信、可亲、可爱三个维度的互动和服务动作。

4. **建立超级用户路径**
   - 完成标准：定义从新用户到超级用户的行为、权益、反馈和贡献路径。

5. **连接复购与转介绍**
   - 完成标准：设计让满意体验、内容、社交证明和推荐自然发生的循环。

6. **设置健康指标**
   - 完成标准：使用复购、互动质量、贡献内容、转介绍、退群/屏蔽率等指标，而非只看人数。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 产品体验和服务交付尚未站稳。
- 用户只想群发促销、薅羊毛裂变或骚扰式触达。
- 私域涉及个人信息、合规和平台规则时缺少审查。

### 作者在书中警告的失败模式

- 把私域当流量池。
- 只追求群规模和触达频次。
- 没有超级用户机制和关系回馈。

### 作者的盲点 / 时代局限

- 平台工具、隐私政策和用户对私域的容忍度会变化。
- 低频、低信任或强监管品类需要更谨慎设计关系边界。
