# 靶心渠道选择

## R — 原文 (Reading)

> “靶心对所有类型的创业公司都适用。”
>
> “大多数创业公司没有认真考虑所有可用渠道。”
>
> 来源：Gabriel Weinberg、Justin Mares《拉新》。

## I — 方法论骨架 (Interpretation)

靶心法的核心不是列渠道清单，而是反舒适区。

团队通常会选择自己熟悉、老板喜欢或竞品正在用的渠道。《拉新》要求先把渠道放进外环，广泛想象每个渠道的可能性；再进入中环，用低成本测试比较获客量、成本和质量；最后只把资源集中到一个最有可能成为主要增长来源的内环渠道。

这个 skill 让 agent 把“我们该投哪里”改写成一个可验证的渠道选择实验。

## A1 — 书中的应用 (Past Application)

### 案例 1：DuckDuckGo

- **问题**：小团队如何在搜索市场获得第一批用户。
- **方法论的使用**：不只盯常规广告，而是测试 PR、SEO、社区等渠道。
- **结论**：早期渠道选择要反直觉地扫描全渠道。
- **结果**：找到与产品特性匹配的拉新路径。

### 案例 2：Mint

- **问题**：金融工具如何在发布前积累用户。
- **方法论的使用**：通过内容、博客和合作渠道提前建立受众。
- **结论**：渠道测试可以早于正式发布。
- **结果**：上线时已有初始需求和传播势能。

### 案例 3：Dropbox

- **问题**：付费获客成本过高。
- **方法论的使用**：从广告转向推荐机制等更匹配的渠道。
- **结论**：最熟悉渠道不一定是核心渠道。
- **结果**：推荐成为可持续拉新杠杆。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 不知道该选哪个获客渠道。
2. 现有渠道成本上升、质量下降或不可规模化。
3. 团队总在熟悉渠道里小修小补。
4. 用户想设计低成本渠道测试。

### 语言信号

- “该选哪个渠道？”
- “获客成本太高怎么办？”
- “我们要不要投短视频/SEO/BD/广告？”
- “怎么用靶心法做渠道测试？”

### 与相邻 skill 的区分

- 与 `growth-readiness-aha-gate` 的区别：本 skill 假设产品已有增长资格。
- 与 `growth-experiment-operating-system` 的区别：本 skill 选渠道，后者管理实验系统。
- 与 `relationship-flywheel-private-domain` 的区别：私域是关系复利渠道，不是所有获客渠道的默认答案。

## E — 可执行步骤 (Execution)

1. **确认增长门禁**
   - 完成标准：说明产品是否已有啊哈时刻、留存或复购证据；没有则先暂停大规模拉新。

2. **列外环渠道**
   - 完成标准：至少列出 8-12 个可能渠道，并包含团队不熟悉但理论可行的渠道。

3. **做粗筛假设**
   - 完成标准：对每个渠道估计用户匹配、成本、速度、可规模化和主要风险。

4. **选择中环测试**
   - 完成标准：选 3 个最有潜力渠道，设计一周到两周内能完成的低成本测试。

5. **比较渠道质量**
   - 完成标准：不仅看线索数量，还看激活率、留存、付费或复购质量。

6. **确定内环核心渠道**
   - 完成标准：给出继续加码、继续测试、放弃或回到外环的判断。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 产品尚未证明核心价值和留存。
- 用户只要求某个已定渠道的广告文案、投放出价或账户操作。
- 渠道选择依赖最新平台价格、政策或实时竞品数据，需要另行查证。

### 作者在书中警告的失败模式

- 只尝试自己熟悉的渠道。
- 同时追太多渠道，无法学到什么。
- 用表面线索量替代真实用户质量。

### 作者的盲点 / 时代局限

- 平台规则和广告价格高度变化，书中渠道案例不能直接套用。
- B2B、强监管或线下行业需要额外检查合规和销售周期。
