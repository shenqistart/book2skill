# 05 增长和流量 Skills

本目录把产品分类 `05_增长和流量` 下的增长、流量、私域、社群、直播电商和流媒体平台书籍，蒸馏成一组可被 agent 调用的分析 skills。

它不是渠道技巧大全，也不是投放建议。它的用途是：当用户面对获客、留存、复购、私域、社群、直播或平台分发问题时，先判断增长问题属于哪一层，再选择合适方法拆解。

## 推荐调用方式

1. 先读 [INDEX.md](./INDEX.md)，按用户问题选择 skill。
2. 如果问题是流量成本高、复购低、口碑弱，优先从 `repeat-customer-experience-loop` 开始。
3. 如果问题是产品还没验证就想做增长，优先使用 `growth-readiness-aha-gate`。
4. 如果问题是团队增长动作零散，使用 `growth-experiment-operating-system`、`north-star-growth-model` 或 `omtm-lean-analytics-focus`。
5. 如果问题是拉新渠道选择，使用 `bullseye-channel-selection`。
6. 如果问题是私域、社群、超级用户或小群传播，使用 `relationship-flywheel-private-domain` 或 `small-group-community-leverage`。
7. 如果问题发生在直播电商、内容平台或流媒体分发中，使用 `live-commerce-fit-map` 或 `platform-power-shift-strategy`。

## 重要边界

- 不提供广告投放报价、平台算法实时判断、GMV 预测或证券投资建议。
- 不把增长实验当成弥补产品价值不足的万能药。
- 不把私域理解为骚扰式触达；关系和信任透支时应停止。
- 不把直播或流媒体平台案例机械套到所有品类。
- 涉及当前平台规则、监管、广告价格和渠道成本时，需要另行查证。
