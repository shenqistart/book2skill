# 交易模型设计

## R — 来源依据 (Reading)

依据“产品即交易”“什么是交易模型”“产品经理为什么要关注交易”等章节。

本仓库不收录原书全文；此处只保留章节级依据和方法论重述。

## I — 方法论骨架 (Interpretation)

- 产品不是孤立物，而是一次广义交易的载体。
- 交易发生需要用户认为主观收益大于成本，企业也能获得可持续回报。
- 产品经理要关注交易成本、相对价格、供需和边际变化。
- 好的产品设计常常是降低交易成本、扩大可交换价值。

## A1 — 书中的应用 (Past Application)

### 案例 1: 链家真房源
- **问题**: 房产交易中最大成本之一是信任和信息真实性。
- **方法论的使用**: 通过真房源降低搜索和信任成本。
- **结论**: 交易成本下降能提高交易效率。
- **结果**: 链家案例被用于说明交易成本比较。

### 案例 2: 拼车价值分配
- **问题**: 乘客、司机、平台之间的价值分配会影响交易成立。
- **方法论的使用**: 用交易模型权衡各方收益和成本。
- **结论**: 分配规则是产品的一部分。
- **结果**: 书中用拼车模式演变说明。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

当用户需要判断一个产品如何在用户和企业之间发生价值交换、交易成本在哪里、交换是否可持续时调用。不适用于只画支付链路。

- "用户为什么愿意交换？"

## E — 可执行步骤 (Execution)

1. **定义交易参与方**
   - 完成标准: 列出用户、企业、第三方及各自交付和获得。

2. **列主观收益与成本**
   - 完成标准: 分别列出效用、金钱、时间、信任、学习成本。

3. **找最大摩擦**
   - 完成标准: 确定阻碍交易发生的首要成本。

4. **设计降低成本动作**
   - 完成标准: 提出减少摩擦或提升效用的产品改动。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 不要把交易模型简化成商业模式收费点。
- 公益、公共服务也有广义交易但目标函数不同。
- 法律和伦理约束不能用交易效率覆盖。

### 作者的盲点 / 时代局限

- 书中大量互联网平台和出行案例，对硬件、医疗、教育等行业需要重新校准。
- 经济学语言很强，可能低估审美、文化和身份认同类产品的非理性价值。
