# 用户价值公式判断

## R — 来源依据 (Reading)

> 原书依据见本节来源说明。

依据“产品经理应追求新体验最大化、旧体验最小化、替换成本最小化”等章节。

本仓库不收录原书全文；此处只保留章节级依据和方法论重述。

## I — 方法论骨架 (Interpretation)

- 用户选择新产品时，不是在真空中评价功能，而是在和旧方案比较。
- 新体验必须大到足以覆盖旧体验损失和替换成本。
- 替换成本包括学习、迁移、信任、社交关系、历史数据等。
- 产品经理要同时增加新价值、保留旧价值、降低切换阻力。

## A1 — 书中的应用 (Past Application)

### 案例 1: 快车动态调价与排队
- **问题**: 供需紧张时不同机制影响乘客和司机体验。
- **方法论的使用**: 比较不同方案的新体验、旧体验和成本。
- **结论**: 没有绝对正确，只有权衡后的价值最大化。
- **结果**: 书中用快车案例展示权衡。

### 案例 2: 乘客修改目的地
- **问题**: 修改目的地看似小功能，会影响司机、乘客、平台。
- **方法论的使用**: 用用户价值和交易成本综合判断。
- **结论**: 功能要考虑完整交易链。
- **结果**: 出行案例展示公式使用。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

当用户要判断一个新产品、新功能或改版是否值得用户替换旧方案时调用，重点比较新体验、旧体验和替换成本。不适用于只修复明确 bug。

### 语言信号

- "用户为什么要换用我们？"
- "这个改版会不会伤害老用户？"
- "新体验够不够抵消迁移成本？"

### 与相邻 skill 的区分

- 本 skill 处理 `用户价值公式判断` 这一单点问题；若问题跨越多个环节，请先读本书 `INDEX.md` 选择组合顺序。

## E — 可执行步骤 (Execution)

1. **列旧体验**
   - 完成标准: 写出用户当前方案中会失去的价值。

2. **列新体验**
   - 完成标准: 写出新方案新增或增强的价值。

3. **估替换成本**
   - 完成标准: 列学习、迁移、信任、关系和风险成本。

4. **判断净价值**
   - 完成标准: 输出净增、持平或净减，并提出降低替换成本方案。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 不要只计算功能数量。
- 强网络效应产品的替换成本可能极高。
- 短期 A/B 数据不能完全覆盖长期信任损失。

### 作者的盲点 / 时代局限

- 书中大量互联网平台和出行案例，对硬件、医疗、教育等行业需要重新校准。
- 经济学语言很强，可能低估审美、文化和身份认同类产品的非理性价值。
