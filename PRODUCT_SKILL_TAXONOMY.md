# 03 如何做产品 — Skill 分类与差异地图

> 这份文档用于回答：这些产品类 skills 是互相冲突，还是互补？应该按什么顺序调用？

## 总判断

这 10 套产品 skill packs 不是彼此替代的一组理论，而是产品判断的不同镜头。

- 《真需求》问：这事真不真？
- 《俞军产品方法论》问：用户价值和交易是否成立？
- 《启示录》问：团队该不该投入资源做？
- 《用户体验要素》问：体验问题发生在哪一层？
- 《简约至上》问：复杂度能不能被删、藏、组织或转移？
- 《微信背后的产品观》问：产品有没有边界、气质和自我演化能力？
- 《上瘾》问：这个产品适不适合形成健康习惯？
- 《任天堂的体验设计》问：用户能否直觉进入、持续获得惊喜和成长？
- 《鞋狗》问：产品如何在真实市场、供应和现金流中活下来？
- 《Little Black Stretchy Pants》问：品牌如何通过门店、社群和文化扩张？

真正的张力不在于谁对谁错，而在于它们默认优化的目标不同。

## 一级分类

| 分类 | 解决的问题 | 对应 skill packs | 典型输出 |
|---|---|---|---|
| 需求与价值判断 | 需求是否真实、价值是否成立、用户是否愿意替换旧方案 | [`true-demand-skill`](./true-demand-skill/), [`yu-jun-product-methodology-skill`](./yu-jun-product-methodology-skill/) | 需求真伪、价值组合、用户价值公式、交易模型 |
| 机会与团队投入 | 这个机会是否值得团队投入，如何在开发前验证 | [`inspired-product-management-skill`](./inspired-product-management-skill/) | 机会评估、产品原则、原型测试、特约用户计划 |
| 体验结构与复杂度 | 体验问题在哪一层，复杂度如何治理 | [`elements-of-user-experience-skill`](./elements-of-user-experience-skill/), [`simple-and-usable-skill`](./simple-and-usable-skill/) | 五层 UX 诊断、范围边界、删除/组织/隐藏/转移策略 |
| 产品边界与增长机制 | 产品是否克制、是否有自然增长和健康习惯循环 | [`wechat-product-philosophy-skill`](./wechat-product-philosophy-skill/), [`hooked-habit-product-skill`](./hooked-habit-product-skill/) | 场景边界、结构演化、自然增长、触发/行动/奖励/投入循环 |
| 体验情绪与持续投入 | 用户是否能自然理解、被惊喜唤醒并感到成长 | [`nintendo-experience-design-skill`](./nintendo-experience-design-skill/) | 直觉体验、惊喜重置、故事成长弧 |
| 市场、品牌与运营现实 | 产品如何在真实市场、供应链、现金流、门店和社群里长出来 | [`shoe-dog-product-building-skill`](./shoe-dog-product-building-skill/), [`little-black-stretchy-pants-skill`](./little-black-stretchy-pants-skill/) | 场地验证、反馈循环、供应风险、品牌运动、零售教育、社群扩张 |

## 差异表

| Skill pack | 核心问题 | 最适合用在 | 默认偏好 | 慎用边界 |
|---|---|---|---|---|
| [`wechat-product-philosophy-skill`](./wechat-product-philosophy-skill/) | 产品是否有边界、气质和自我演化能力 | 社交、内容、消费级产品、长期产品观 | 克制、自然增长、场景边界 | 不要把微信案例机械套到 B2B、专业工具或监管场景 |
| [`true-demand-skill`](./true-demand-skill/) | 需求是不是真的，价值组合是否成立 | 0 到 1、定位、需求判断 | 求真、价值、共识、场景 | 不能用概念包装替代真实验证 |
| [`yu-jun-product-methodology-skill`](./yu-jun-product-methodology-skill/) | 用户价值和交易模型是否成立 | 产品决策、商业模型、替代旧方案 | 效用、成本、边际、交易 | 不要把理性公式当成唯一判断，需补充人性和场景 |
| [`inspired-product-management-skill`](./inspired-product-management-skill/) | 团队是否该投入资源做这个机会 | 产品团队、敏捷、机会评估、原型验证 | 先验证，再开发 | 不适合把早期混沌创业直接流程化 |
| [`elements-of-user-experience-skill`](./elements-of-user-experience-skill/) | 体验问题发生在哪一层 | UX 诊断、信息架构、页面流程 | 战略到表现的分层拆解 | 不要跳过战略层直接修 UI |
| [`simple-and-usable-skill`](./simple-and-usable-skill/) | 怎么降低复杂度 | 功能膨胀、界面复杂、流程太重 | 删除、组织、隐藏、转移 | 不能删除安全、合规、可解释性等必要复杂度 |
| [`hooked-habit-product-skill`](./hooked-habit-product-skill/) | 产品能否形成健康使用习惯 | 高频产品、留存、触发机制 | 触发、行动、奖励、投入 | 必须做操控伦理筛查，不适合低频严肃任务硬做上瘾 |
| [`nintendo-experience-design-skill`](./nintendo-experience-design-skill/) | 用户如何自然理解、惊喜、成长 | 游戏、教育、互动体验、onboarding | 直觉、惊喜、故事成长 | 不要用惊喜破坏效率、信任或专业任务稳定性 |
| [`shoe-dog-product-building-skill`](./shoe-dog-product-building-skill/) | 产品如何在真实市场里活下来 | 创业、实体产品、供应链、现金流 | 场地验证、反馈循环、生存纪律 | 不要用创业故事替代财务和供应链硬约束 |
| [`little-black-stretchy-pants-skill`](./little-black-stretchy-pants-skill/) | 品牌如何通过门店、社群、文化扩张 | 消费品牌、零售、生活方式品牌 | 社群、教育、文化、直营反馈 | 不要用文化叙事掩盖质量、价格或品类不成立 |

## 推荐调用顺序

### 新产品从 0 到 1

1. [`true-demand-skill`](./true-demand-skill/)：确认需求、价值、共识和场景是否真实。
2. [`yu-jun-product-methodology-skill`](./yu-jun-product-methodology-skill/)：计算用户价值、替换成本和交易模型。
3. [`inspired-product-management-skill`](./inspired-product-management-skill/)：做机会评估、产品原则和开发前验证。
4. [`elements-of-user-experience-skill`](./elements-of-user-experience-skill/)：定位体验问题属于战略、范围、结构、框架还是表现层。
5. [`simple-and-usable-skill`](./simple-and-usable-skill/)：治理复杂度，决定删除、组织、隐藏或转移。
6. [`wechat-product-philosophy-skill`](./wechat-product-philosophy-skill/)：检查场景边界、产品结构、自然增长和产品气质。
7. [`hooked-habit-product-skill`](./hooked-habit-product-skill/)：只有当产品适合高频习惯时，再设计触发、行动、奖励和投入。
8. [`nintendo-experience-design-skill`](./nintendo-experience-design-skill/)：优化直觉进入、惊喜和成长感。
9. [`shoe-dog-product-building-skill`](./shoe-dog-product-building-skill/) 或 [`little-black-stretchy-pants-skill`](./little-black-stretchy-pants-skill/)：当产品涉及实体、品牌、零售、社群或供应链时再调用。

### 按问题入口选择

| 你的问题 | 优先调用 | 可组合调用 |
|---|---|---|
| 不确定需求是不是真的 | `true-demand-skill` | `yu-jun-product-methodology-skill` |
| 不确定用户会不会换掉旧方案 | `yu-jun-product-methodology-skill` | `true-demand-skill` |
| 团队争论要不要投入开发 | `inspired-product-management-skill` | `true-demand-skill`, `yu-jun-product-methodology-skill` |
| 体验很乱但不知道哪里坏了 | `elements-of-user-experience-skill` | `simple-and-usable-skill` |
| 功能太多、页面太复杂 | `simple-and-usable-skill` | `wechat-product-philosophy-skill` |
| 留存低、想做习惯机制 | `hooked-habit-product-skill` | `wechat-product-philosophy-skill`, `nintendo-experience-design-skill` |
| 产品越来越像功能集合 | `wechat-product-philosophy-skill` | `simple-and-usable-skill`, `elements-of-user-experience-skill` |
| onboarding 或互动体验不自然 | `nintendo-experience-design-skill` | `elements-of-user-experience-skill` |
| 产品要到真实市场卖 | `shoe-dog-product-building-skill` | `true-demand-skill`, `yu-jun-product-methodology-skill` |
| 做消费品牌、门店、社群 | `little-black-stretchy-pants-skill` | `shoe-dog-product-building-skill`, `wechat-product-philosophy-skill` |

## 主要张力与调和方式

| 张力 | 表面冲突 | 调和方式 |
|---|---|---|
| 微信 vs 上瘾 | 微信偏克制，上瘾偏触发 | 先用微信判断是否应该打扰用户，再用上瘾设计健康触发，并用伦理筛查收尾 |
| 简约至上 vs 用户体验要素 | 简约想删，UX 要完整分层 | 先用 UX 找到问题层，再决定删、藏、组织或转移 |
| 真需求 vs 品牌运动 | 真需求求事实，品牌运动讲共识和情绪 | 先确认需求和价值真实，再决定是否放大成品牌叙事 |
| 俞军 vs 微信 | 俞军偏理性效用，微信偏人性和气质 | 一个算价值交换，一个看长期产品人格；先算成立，再看是否像一个完整产品 |
| 启示录 vs 鞋狗 | 启示录偏流程验证，鞋狗偏创业现场感 | 团队成熟用启示录治理流程，早期混沌用鞋狗补真实市场和现金流压力 |
| 任天堂 vs 简约至上 | 任天堂追求惊喜，简约追求降低负担 | 基础任务先简约，关键体验节点再加入惊喜和成长感 |

## 不建议的混用方式

- 不要在需求未证实时直接调用品牌、习惯或体验情绪类 skills。
- 不要用 `hooked-habit-product-skill` 给低频、高风险、严肃任务强行加触发和奖励。
- 不要用 `wechat-product-philosophy-skill` 的克制来否定合规、安全、无障碍和专业工具需要的明确提示。
- 不要用 `simple-and-usable-skill` 删除审计、权限、解释、错误恢复等必要复杂度。
- 不要用 `little-black-stretchy-pants-skill` 的文化系统掩盖产品质量或品类价值不成立。

## 最短决策树

```text
这是一个需求真假问题吗？
  是 -> true-demand-skill -> yu-jun-product-methodology-skill
  否 -> 继续

这是一个团队投入和验证流程问题吗？
  是 -> inspired-product-management-skill
  否 -> 继续

这是一个体验结构或复杂度问题吗？
  是 -> elements-of-user-experience-skill -> simple-and-usable-skill
  否 -> 继续

这是一个增长、留存、习惯或产品边界问题吗？
  是 -> wechat-product-philosophy-skill -> hooked-habit-product-skill
  否 -> 继续

这是一个互动体验、学习曲线或情绪持续问题吗？
  是 -> nintendo-experience-design-skill
  否 -> 继续

这是一个实体产品、品牌、门店、社群或供应链问题吗？
  是 -> shoe-dog-product-building-skill -> little-black-stretchy-pants-skill
```
