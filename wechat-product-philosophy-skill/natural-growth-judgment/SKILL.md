# 自然增长判断

## R — 来源

> 原书依据见本节来源说明。

依据书中关于自然增长、KPI 是副产品、未到爆发点不推广、避免战略行为替代真实需求、从人出发改变商业模式等论述。

## I — 方法论骨架

推广不能替代产品生命力。
如果产品没有自然增长迹象，导流和广告只会把用户更快送进不成熟体验，并提高下一次拉动成本。
KPI、整合、打通、多平台、导入导出常常来自组织战略，而不是用户真实意愿。
增长动作应该放大已经存在的自发行为，而不是制造用户并不想要的行为。
商业模式也应从用户怎样自然动起来开始，而不是先画战略图再要求用户配合。

## A1 — 书中的应用

### 案例: 微信早期推广节奏

- **问题**: 是否利用腾讯资源快速推广微信。
- **方法论的使用**: 作者先观察自然增长曲线，而不是一开始就投放广告。
- **结论**: 产品没到爆发点时推广收益不成比例。
- **结果**: 推广被视为放大器，而非产品生命力来源。

### 案例: 反对简单整合和导入

- **问题**: 是否把其他产品内容导入朋友圈。
- **方法论的使用**: 作者检查导入内容是否符合朋友圈氛围和用户意愿。
- **结论**: 不合场景的导入会破坏价值。
- **结果**: 反对为战略整合牺牲真实场景。

## A2 — 触发场景

1. 用户想用投放、导流或资源位拉新。
2. 用户想把多个产品打通、整合、合并或互相导入。
3. 用户讨论 KPI 压力和产品体验冲突。
4. 用户想判断产品是否已经有自然增长信号。

### 语言信号

- “该不该推广”
- “先导一批流量”
- “KPI 压力”
- “把几个产品打通”
- “多平台/全面化”
- “自然增长曲线”

### 与相邻 skill 的区分

- 与 `product-structure-evolution` 的区别: 本 skill 判断增长动作是否应该发生，后者判断结构是否能承载。
- 与 `demand-psychology-mining` 的区别: 本 skill 聚焦增长和组织动作，后者聚焦单个需求。

## E — 可执行步骤

1. **确认自然增长信号**
   - 完成标准: 判断是否已有自发留存、主动邀请、口碑表达、重复使用或小范围扩散。

2. **识别增长动作来源**
   - 完成标准: 标明动作来自用户意愿、产品生命力、商业战略、KPI 压力还是组织协同。

3. **检查放大副作用**
   - 完成标准: 评估流量进入后是否会造成差体验、氛围破坏、关系链污染、复杂度上升或信任下降。

4. **输出增长决策**
   - 完成标准: 给出 `先不推 / 小范围验证 / 放大自然行为 / 修产品后再推 / 拒绝战略整合`。

## B — 边界

- 成熟产品的增长实验不应被一概否定。
- 某些公共服务、合规迁移、平台生态建设可能需要非自然启动。
- 不要把自然增长浪漫化，留存、转化和用户质量仍需数据验证。
- B2B 产品的增长路径可能不是大众口碑扩散。

### 失败模式

- 产品没准备好就导入大流量。
- 为 KPI 改变核心体验。
- 50 分产品互相整合，以为能变成 80 分。
- 把广告、导流、打通当作需求本身。
