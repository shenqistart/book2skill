# 《微信背后的产品观》案例候选

> 来源说明见 [../source/SOURCE.md](../source/SOURCE.md)。本文件记录案例类型及其方法论用途，不复刻原文。

## 通过案例

### 1. 漂流瓶、摇一摇、附近的人

- **对应问题**: 看似轻量的功能为什么能触发用户行动。
- **抽象用途**: 说明心理诉求、偶遇感、存在感和群体行为如何生成需求。
- **进入 skills**: `human-group-sensing`、`demand-psychology-mining`、`scenario-boundary-design`。

### 2. 语音消息

- **对应问题**: 低成本输入和真实沟通场景如何改变用户行为。
- **抽象用途**: 说明功能价值来自场景匹配，而不是技术炫耀。
- **进入 skills**: `scenario-boundary-design`、`product-structure-evolution`。

### 3. 朋友圈

- **对应问题**: 熟人关系链中的表达、观看和互动为什么需要强边界。
- **抽象用途**: 说明群体行为、社交压力、默认可见性和产品克制之间的关系。
- **进入 skills**: `human-group-sensing`、`product-structure-evolution`、`product-spirit-expression`。

### 4. Web/PC 微信相关讨论

- **对应问题**: 多端、整合和便利性并不天然等于更好的产品。
- **抽象用途**: 说明产品边界、主场景和结构一致性优先于全面覆盖。
- **进入 skills**: `scenario-boundary-design`、`natural-growth-judgment`。

### 5. 欢迎页、文案和 UI 细节

- **对应问题**: 产品价值观如何被用户感知。
- **抽象用途**: 说明气质不是装饰，而是通过具体表达持续建立信任。
- **进入 skills**: `product-spirit-expression`。
