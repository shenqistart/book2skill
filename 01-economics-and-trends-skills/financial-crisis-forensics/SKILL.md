# 金融危机法医复盘

## R — 原文 (Reading)

> “七大祸源包括：两个泡沫、过度杠杆、松散的金融监管、金融工程复杂性、评级机构、疯狂的金融激励机制。”
>
> 来源：布林德《当音乐停止之后》相关章节。

## I — 方法论骨架 (Interpretation)

金融危机不是“某一天突然坏掉”。

它通常有五层链条：资产泡沫提供燃料，杠杆放大收益和损失，期限错配制造流动性脆弱，监管和评级让风险被低估，触发事件把不信任扩散成恐慌。
救助政策也要分层看：流动性支持、资本注入、资产处置、担保、财政刺激和监管修复不是同一件事。
复盘危机时，不能只找罪人或单一事件，要重建“脆弱性 -> 触发 -> 传播 -> 政策响应 -> 后遗症”的链条。

这个 skill 的目标是做危机法医，不是做市场恐吓。

## A1 — 书中的应用 (Past Application)

### 案例 1：布林德的七大祸源

- **问题**：2008 年危机为什么不是单点事故。
- **方法论的使用**：把房地产泡沫、债券泡沫、杠杆、监管、次贷、金融工程、评级和激励逐项拆开。
- **结论**：雷曼是触发和放大节点，不是全部原因。
- **结果**：复盘从叙事转为因果链。

### 案例 2：货币基金和 AIG 恐慌

- **问题**：为什么非银行机构也会造成系统性风险。
- **方法论的使用**：检查短债长投、担保承诺、衍生品敞口和赎回压力。
- **结论**：影子银行也能发生类似银行挤兑的恐慌。
- **结果**：金融稳定分析不能只看商业银行。

### 案例 3：危机后的监管钟摆

- **问题**：危机后监管为何容易过紧或被套利。
- **方法论的使用**：观察危机记忆、公众愤怒、行业游说和监管执行。
- **结论**：监管是动态博弈，过松和过紧都会制造新问题。
- **结果**：后危机分析必须关注制度修复是否真正落地。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户问某个市场是否存在泡沫、杠杆、流动性或系统性风险。
2. 用户想复盘一次金融危机、银行挤兑、地产链条风险或影子银行事件。
3. 用户把危机归因于单个坏人、单个机构或单个政策。
4. 用户需要评估救助、担保、注资、降息、监管改革分别解决什么问题。

### 语言信号

- “这会不会引发金融危机？”
- “雷曼式风险在哪里？”
- “为什么救助看起来不公平但还要做？”
- “泡沫和杠杆怎么传导成恐慌？”

### 与相邻 skill 的区分

- 与 `monetary-regime-map` 的区别：本 skill 复盘危机链条；央行工具和货币制度由后者解释。
- 与 `macro-constraint-map` 的区别：本 skill 聚焦金融脆弱性，不覆盖全部宏观结构问题。

## E — 可执行步骤 (Execution)

1. **识别脆弱性**
   - 完成标准：列出资产泡沫、杠杆、期限错配、集中敞口、影子银行、监管缺口。

2. **识别触发事件**
   - 完成标准：区分真正原因、直接触发和传播节点。

3. **重建传播链**
   - 完成标准：说明损失如何从资产价格传到资产负债表、流动性、信心和实体经济。

4. **拆分政策响应**
   - 完成标准：分别评估流动性、资本、担保、资产处置、财政刺激、监管修复。

5. **输出危机法医报告**
   - 完成标准：给出“已有证据、缺失证据、可能链条、误判风险、不可下结论项”。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 只是普通市场回撤、单个资产下跌或短期情绪波动。
- 用户要求投资建议、做空建议、资产配置或具体交易。
- 没有金融传导链条和资产负债表信息。

### 作者在书中警告的失败模式

- 只看雷曼等单一事件，忽略危机前的泡沫和杠杆。
- 相信金融机构可以完全自我监管。
- 危机后太快遗忘，导致监管被重新放松或被套利。

### 作者的盲点 / 时代局限

- 2008 年美国危机经验不等于所有金融体系的危机模板。
- 公开复盘材料可能低估政治协商、内部约束和无法公开的信息。
