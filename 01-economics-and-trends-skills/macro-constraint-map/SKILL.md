# 中国宏观约束地图

## R — 原文 (Reading)

> “这份讲义则更多在讲‘宏观经济理论怎么用’。”
>
> “供给侧结构性改革一定是基础性制度的改革，一定是体制、机制的改革。”
>
> 来源：徐高《宏观经济学二十五讲：中国视角》导论；黄奇帆《分析与思考》。

## I — 方法论骨架 (Interpretation)

宏观问题不是把几个指标排在一起。

这个 skill 先识别经济运行的“容器”：金融结构、财政关系、房地产机制、资本市场、地方激励、外需和产业结构。
然后把问题放进四类约束：需求约束、供给约束、金融约束、制度约束。
同一个货币或财政动作，在不同容器里会有不同传导。
中国宏观分析尤其要检查债务和资产价格是否卡住传导，结构改革是否能释放长期供给。

它的产出不是预测，而是一张约束关系图。

## A1 — 书中的应用 (Past Application)

### 案例 1：徐高的“中国视角”

- **问题**：为什么不能直接照搬西方宏观教材。
- **方法论的使用**：用主流宏观语言解释中国现实，同时识别市场分割、预算软约束、货币传导阻塞等特殊约束。
- **结论**：规律是一般的，但容器不同，现象不同。
- **结果**：形成“模型加约束”的宏观分析方式。

### 案例 2：黄奇帆的企业去杠杆

- **问题**：如何降低宏观杠杆率。
- **方法论的使用**：区分政府、居民、企业部门，指出中国关键在企业债务和融资结构。
- **结论**：提高直接融资和资本市场质量，是比猛烈收缩信贷更稳的路径。
- **结果**：把去杠杆从口号转成融资结构改革。

### 案例 3：房地产基础制度

- **问题**：房地产为何牵动宏观。
- **方法论的使用**：把土地、融资、地方财力、居民负债和城市化放在同一约束图中。
- **结论**：房地产不是单一行业，而是财政、金融和居民资产负债表的交叉点。
- **结果**：提示 agent 不要用简单供需解释宏观房地产问题。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户问中国债务、房地产、地方财政、货币宽松、内需不足、产能、通缩或结构改革。
2. 用户试图用单一指标解释宏观问题。
3. 用户讨论短期刺激和长期改革之间的取舍。
4. 用户需要把宏观叙事转成约束关系，而非下预测结论。

### 语言信号

- “中国宏观现在到底卡在哪里？”
- “为什么降息/放水没有明显效果？”
- “房地产和地方债是什么关系？”
- “这个政策是需求刺激还是供给侧改革？”

### 与相邻 skill 的区分

- 与 `monetary-regime-map` 的区别：本 skill 看宏观约束全图；央行工具、预期和货币锚交给后者。
- 与 `china-reform-path` 的区别：本 skill 判断宏观传导；制度改革的历史阶段和执行路径交给后者。

## E — 可执行步骤 (Execution)

1. **定义宏观问题**
   - 完成标准：把问题归为增长、就业、通胀、债务、房地产、外需、产业或财政中的一类或多类。

2. **画四类约束**
   - 完成标准：分别列出需求约束、供给约束、金融约束、制度约束。

3. **定位传导堵点**
   - 完成标准：说明政策工具从中央到企业、居民、地方和金融机构的传导路径，标出堵点。

4. **区分短期和长期**
   - 完成标准：把需求管理、风险处置、结构改革和制度红利分开。

5. **输出约束地图**
   - 完成标准：用“主约束、次约束、关键假设、不可判断项”四栏输出。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户要实时宏观数据、政策原文或最新监管变化。
- 用户要求股市、房价、汇率、利率路径预测。
- 用户只问局部市场的税收或价格政策，应转 `market-policy-incidence`。

### 作者在书中警告的失败模式

- 把宏观理论当成抽象模型，不看现实约束。
- 把去杠杆等同于简单压缩信贷，忽略融资结构和资本市场。
- 把供给侧结构性改革误解为短期需求管理。

### 作者的盲点 / 时代局限

- 政策类文本可能低估执行偏差和地方差异。
- 教材化框架可能无法覆盖突发外部冲击和地缘政治因素。
