# 运动员反馈产品循环

## R — 来源依据 (Reading)

依据鲍尔曼轻便跑鞋、华夫鞋底、运动员试穿和产品召回等章节。

本仓库不收录原书全文；此处只保留章节级依据和方法论重述。

## I — 方法论骨架 (Interpretation)

- 运动产品的创新来自真实身体反馈。
- 专业用户能暴露普通测试看不到的性能问题。
- 反馈循环必须连接设计、材料、供应和召回，而不只是收集意见。

## A1 — 书中的应用 (Past Application)

### 案例 1: 鲍尔曼华夫鞋底实验
- **问题**: 如何让跑鞋更轻更有抓地力。
- **方法论的使用**: 把教练和运动员经验转化为材料实验。
- **结论**: 专业场景推动产品突破。
- **结果**: 形成 Nike 早期产品差异。

### 案例 2: LD-1000 召回
- **问题**: 产品设计存在伤害风险。
- **方法论的使用**: 承认质量问题并召回。
- **结论**: 速度不能压过安全。
- **结果**: 品牌信任得以维护。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

当产品强依赖真实高强度使用体验，需要把专业用户、试穿、材料实验和快速改进连成循环时调用。不适用于低风险装饰性改动。

### 语言信号

- "专业用户反馈怎么用？"
- "产品体验需要实测"
- "我们要不要召回或改材料？"

### 与相邻 skill 的区分

- 本 skill 处理 `运动员反馈产品循环` 这一单点问题；若问题跨越多个环节，请先读本书 `INDEX.md` 选择组合顺序。

## E — 可执行步骤 (Execution)

1. **选择极端用户**
   - 完成标准: 找高频、高强度、能表达体验的用户。

2. **让反馈进入设计**
   - 完成标准: 把反馈转成材料、结构、尺码、工艺问题。

3. **小批量验证**
   - 完成标准: 用小批样品反复试穿或试用。

4. **建立质量停线**
   - 完成标准: 出现伤害或信任风险时停止销售或召回。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 极端用户不是全部用户。
- 不能让明星意见压过安全数据。

### 作者的盲点 / 时代局限

- 自传带有强烈创始人视角，对失败决策可能有事后合理化。
- Nike 早期处在跑步热潮和全球贸易窗口，今天不可机械复制。
