# 供应依赖风险拆解

## R — 来源依据 (Reading)

依据蓝带体育与鬼冢虎代理关系破裂、寻找替代工厂和 Nike 品牌诞生的章节。

本仓库不收录原书全文；此处只保留章节级依据和方法论重述。

## I — 方法论骨架 (Interpretation)

- 代理和渠道红利会掩盖控制权风险。
- 当核心供给不在自己手里，增长越快，受制于人的风险越大。
- 自有品牌或多源供应不是面子工程，而是生存选项。

## A1 — 书中的应用 (Past Application)

### 案例 1: 鬼冢虎代理权之争
- **问题**: 蓝带依赖鬼冢虎供货。
- **方法论的使用**: 识别代理关系被切断的生存风险。
- **结论**: 必须发展自有品牌和替代供应。
- **结果**: Nike 因危机被迫诞生。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

当产品业务依赖单一供应商、渠道、平台或授权方，且对方可能改变条件时调用。不适用于已经有多源稳定供应的小采购问题。

### 语言信号

- "我们太依赖一个供应商"
- "平台规则一变就完了"
- "要不要做自有品牌？"

### 与相邻 skill 的区分

- 本 skill 处理 `供应依赖风险拆解` 这一单点问题；若问题跨越多个环节，请先读本书 `INDEX.md` 选择组合顺序。

## E — 可执行步骤 (Execution)

1. **列依赖点**
   - 完成标准: 列出供应、渠道、IP、数据、支付等单点依赖。

2. **评估对方动机**
   - 完成标准: 判断对方是否有绕开、涨价、断供或复制的动机。

3. **设计备份路径**
   - 完成标准: 准备多源供应、自有品牌、替代渠道或协议保护。

4. **定义触发阈值**
   - 完成标准: 约定何时启动替代路径。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 自建供应链成本很高，不要因恐惧过早重资产化。
- 合同问题需法律审查。

### 作者的盲点 / 时代局限

- 自传带有强烈创始人视角，对失败决策可能有事后合理化。
- Nike 早期处在跑步热潮和全球贸易窗口，今天不可机械复制。
