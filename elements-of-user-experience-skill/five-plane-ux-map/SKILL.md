# 五层体验地图

## R — 来源依据 (Reading)

> 原书依据见本节来源说明。

依据用户体验五层模型。

本仓库不收录原书全文；此处只保留章节级依据和方法论重述。

## I — 方法论骨架 (Interpretation)

- 用户体验由五个层面共同构成。
- 越底层的问题越会影响上层。
- 诊断时要先定位层级，再谈解决方案。

## A1 — 书中的应用 (Past Application)

### 案例 1: 视频网站示例
- **问题**: 同一个页面问题可能来自目标、功能、结构或视觉。
- **方法论的使用**: 用五层拆解。
- **结论**: 避免只在表现层修补。
- **结果**: 体验问题被定位。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

当用户面对一个模糊的体验问题，不知道是战略、范围、结构、框架还是表现层出错时调用。不适用于只改颜色字号。

### 语言信号

- "这个 UX 问题到底是哪层？"
- "页面不好用但不知道原因"
- "用户体验怎么系统分析？"

### 与相邻 skill 的区分

- 本 skill 处理 `五层体验地图` 这一单点问题；若问题跨越多个环节，请先读本书 `INDEX.md` 选择组合顺序。

## E — 可执行步骤 (Execution)

1. **列五层现状**
   - 完成标准: 写出战略、范围、结构、框架、表现。

2. **定位断点**
   - 完成标准: 找最早不清或冲突的层。

3. **给修复顺序**
   - 完成标准: 从底层到上层安排改进。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 不要把模型当瀑布流程。
- 服务体验和线下体验需补充接触点分析。

### 作者的盲点 / 时代局限

- 模型源自 Web 与软件体验，现代多端、AI 和服务体验需扩展。
- 五层容易被误解成瀑布流程。
