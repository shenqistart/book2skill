# 价值组合诊断

## R — 来源依据 (Reading)

> 原书依据见本节来源说明。

依据“功能价值与效率需求”“情绪价值与情感需求”“资产价值与投资需求”“产品创新——价值组合”“品牌价值”等章节。

本仓库不收录原书全文；此处只保留章节级依据和方法论重述。

## I — 方法论骨架 (Interpretation)

- 需求不是一个单点功能，而是一组价值材料的组合。
- 功能价值回答效率问题，情绪价值回答感受和身份问题，资产价值回答未来收益问题，品牌价值回答长期识别和存在姿态。
- 很多产品陷入同质化，是因为只在功能层加码，却没有改变价值组合。
- 诊断产品时，应先拆开价值来源，再判断哪些价值可叠加、哪些会互相冲突。
- 真正的创新常来自价值组合方式变化，而不只是新增功能。

## A1 — 书中的应用 (Past Application)

### 案例 1: 供应链代际案例
- **问题**: 如何解释中国从丝绸之路、果链特链到 SHEIN 的供应链优势。
- **方法论的使用**: 用功能价值和效率需求看不同供应链如何降低成本、提升响应速度。
- **结论**: 效率价值可以成为产品创新的底座。
- **结果**: 供应链能力改变了产品价值结构。

### 案例 2: 品牌与白牌区分
- **问题**: 白牌和品牌都能卖货，但长期价值不同。
- **方法论的使用**: 把白牌视为供应链成熟和渠道重构的结果，把品牌视为情绪与识别资产。
- **结论**: 品牌不是贴标，而是稳定的价值唤起。
- **结果**: 可口可乐、网红品牌、大牌案例被用于说明品牌价值差异。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

当用户想判断一个产品到底满足功能需求、情绪需求、资产需求还是品牌需求，或想重新组合产品价值以避免只卷功能时调用。不适用于单纯竞品功能罗列或已经确定的工程实现。

### 语言信号

- "这个需求到底值不值钱？"
- "我们是不是只是在堆功能？"
- "用户为什么愿意为这个付费？"
- "这个产品有功能价值还是情绪价值？"

## E — 可执行步骤 (Execution)

1. **拆价值类型**
   - 完成标准: 列出产品当前提供的功能、情绪、资产、品牌四类价值。

2. **找主价值与副价值**
   - 完成标准: 判断哪一种价值是购买理由，哪几种只是锦上添花。

3. **检查组合冲突**
   - 完成标准: 标出会互相抵消的价值，比如极致低价与高端身份感。

4. **提出重组方案**
   - 完成标准: 输出 1-3 个可测试的价值组合假设。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 不要把所有需求都包装成情绪价值。
- 不替代真实用户访谈、价格测试和供给能力评估。
- B2B 产品的资产价值与风险约束要单独建模。

### 作者的盲点 / 时代局限

- 书中案例大量来自消费、手机、新消费和平台语境，B2B、深科技、监管产品需要另加约束。
- 价值和共识框架解释力强，但在早期极小样本阶段容易被过度概念化。
