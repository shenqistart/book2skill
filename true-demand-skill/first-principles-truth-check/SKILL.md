# 第一性原理求真检查

## R — 来源依据 (Reading)

依据“应然与实然”“第一性原理”“后记 造物之心”等章节。

本仓库不收录原书全文；此处只保留章节级依据和方法论重述。

## I — 方法论骨架 (Interpretation)

- 创造者最容易把“我希望世界怎样”误当成“世界实际怎样”。
- 求真是把应然愿望、行业二手结论和市场噪音剥离，回到事实、约束、因果和可验证假设。
- 第一性原理不是装深刻，而是在关键处重新推导：这个需求为什么存在，价值从哪里来，约束是什么，什么不能被绕过。
- 一个好产品既要有愿景，也要服从实然。

## A1 — 书中的应用 (Past Application)

### 案例 1: 苹果气质与有所不为
- **问题**: 苹果式产品判断容易被外界学成表面审美。
- **方法论的使用**: 回到产品气质、边界和基础设施条件，而不是照抄样式。
- **结论**: 求真要求识别真正因果。
- **结果**: 案例用于说明“有所为有所不为”。

### 案例 2: 山寨手机时代
- **问题**: 山寨看似满足需求，但会受制于基础设施和正规化趋势。
- **方法论的使用**: 用实然条件看模式边界。
- **结论**: 邪不压正不是道德判断，而是系统约束变化。
- **结果**: 传音等案例展示转型正规军的路径。

## A2 — 触发场景 (Future Trigger)

## E — 可执行步骤 (Execution)

1. **拆应然**
   - 完成标准: 写出团队希望成立的判断和背后的愿望。

2. **列实然**
   - 完成标准: 列出现有事实、硬约束、不可绕过的成本和反证。

3. **重推因果**
   - 完成标准: 从事实出发重建需求、价值、模式之间的因果链。

4. **输出保留/修正/放弃**
   - 完成标准: 给出结论，并标注还缺什么证据。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 不要把第一性原理当作反对经验的借口。
- 事实不足时只能输出假设，不要装成确定结论。
- 强监管、医学、金融等场景需专业审查。

### 作者的盲点 / 时代局限

- 书中案例大量来自消费、手机、新消费和平台语境，B2B、深科技、监管产品需要另加约束。
- 价值和共识框架解释力强，但在早期极小样本阶段容易被过度概念化。
