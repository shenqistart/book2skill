# 模式成立性检查

## R — 来源依据 (Reading)

依据“能力系统”“变现逻辑”“分配机制”等章节。

本仓库不收录原书全文；此处只保留章节级依据和方法论重述。

## I — 方法论骨架 (Interpretation)

- 需求成立不代表模式成立。
- 模式要回答四个硬问题：需要什么能力系统，拿谁的钱，如何变现，如何分配。
- 能力系统不是资源罗列，而是持续交付价值所需的组织、供应、技术、渠道和认知组合。
- 变现和分配如果不能承接价值创造，产品会在增长后崩掉。

## A1 — 书中的应用 (Past Application)

### 案例 1: 瑞幸能力系统
- **问题**: 咖啡产品看似简单，但增长背后是门店、供应链、数字化和营销能力。
- **方法论的使用**: 用能力系统解释新模式为何能跑通。
- **结论**: 产品机会需要配套能力。
- **结果**: 瑞幸案例用于说明新能力如何构建。

### 案例 2: SHEIN 供应链模式
- **问题**: 快时尚机会不是只靠前端卖货。
- **方法论的使用**: 用小单快反、供应链和平台化能力看模式。
- **结论**: 能力系统决定价值组合的上限。
- **结果**: 模式不是一句定位，而是经营结构。

## E — 可执行步骤 (Execution)

1. **画能力系统**
   - 完成标准: 列出交付该价值必须稳定具备的 3-7 个能力。

2. **回答拿谁的钱**
   - 完成标准: 明确付款人、使用者、受益者是否一致。

3. **检查变现路径**
   - 完成标准: 说明收入发生在价值链哪个环节，是否可重复。

4. **检查分配机制**
   - 完成标准: 判断关键参与方是否有持续贡献的激励。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 不要用融资故事替代变现逻辑。
- 能力系统缺失时，不要把问题归因于用户不懂。
- 早期探索可保留假设，但必须标明证据等级。

### 作者的盲点 / 时代局限

- 书中案例大量来自消费、手机、新消费和平台语境，B2B、深科技、监管产品需要另加约束。
- 价值和共识框架解释力强，但在早期极小样本阶段容易被过度概念化。
