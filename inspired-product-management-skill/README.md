    # 《启示录：打造用户喜爱的产品》Skills for AI Agents

    > 用人员、流程和产品三条线，建立能探索有价值、可用、可行产品的现代产品管理体系。

    ## 这套 Skills 能解决什么问题？

    | 使用场景 | 调用 Skill |
    |---|---|
    | 当团队有产品想法但还没决定是否投入，需要先判断问题、市场、竞争、时机和成功条件时调用。 | [`opportunity-assessment-ten-questions`](opportunity-assessment-ten-questions/SKILL.md) |
| 当团队做取舍时缺少一致价值标准，或者不同部门各按自己的偏好推进功能时调用。 | [`product-principles-decision-frame`](product-principles-decision-frame/SKILL.md) |
| 当产品需要深度目标用户参与定义、测试和推荐，但又不能被单一客户定制牵着走时调用。 | [`charter-user-program`](charter-user-program/SKILL.md) |
| 当团队想在开发前验证产品价值和可用性，用原型代替厚文档沟通时调用。 | [`high-fidelity-prototype-testing`](high-fidelity-prototype-testing/SKILL.md) |
| 当团队使用敏捷却发现产品探索、设计和开发混在一起，导致迭代只是快速做错东西时调用。 | [`product-software-agile-adaptation`](product-software-agile-adaptation/SKILL.md) |
