# 工具与器具系统

## R — 来源

源文件: `chajing-skill/source/chajing.md`。

依据《二之具》的采制工具、《四之器》的煮饮器物，以及《九之略》关于哪些场景可废哪些器具的说明。

## I — 方法论骨架

《茶经》里的器具不是摆设清单，而是一套功能分工。
制茶之具服务于采摘、蒸制、成形、焙干、封藏。
煮饮之器服务于起火、支锅、滤水、炙茶、碾末、量末、熟水、分盏、清洁和收纳。
判断器具是否必要，要先问它承担什么功能，而不是问它是否古雅。
在山野环境可以用环境替代部分器具；在城邑王公之门，器具缺失反而会破坏完整茶事。

## A1 — 书中的应用

### 案例: 二十四器与九之略
- **问题**: 为什么《茶经》列出大量器具，又允许部分省略。
- **方法论的使用**: 作者在《四之器》列完整系统，在《九之略》按场景说明可省项。
- **结论**: 器具完整性取决于场景和功能替代。
- **结果**: 形成“功能优先于名物”的配置逻辑。

## A2 — 触发场景

1. 用户要配置《茶经》茶席或教学器具。
2. 用户问某件器具在流程中的作用。
3. 用户要区分制茶工具和煮饮器具。

### 语言信号

- “《茶经》需要哪些茶具”
- “风炉、鍑、则分别做什么”
- “二之具和四之器有什么区别”
- “茶席器具怎么配”

### 与相邻 skill 的区分

- 与 `water-fire-brewing-method` 的区别: 本 skill 配器具；煮饮动作由水火 skill 处理。
- 与 `contextual-simplification` 的区别: 本 skill 给完整功能系统；简化判断另行处理。

## E — 可执行步骤

1. **识别任务类型**
   - 完成标准: 判断是制茶、煮饮、展示、教学、户外还是收藏讨论。

2. **按功能分组器具**
   - 完成标准: 分为采制、火、水、茶末、分饮、清洁、收纳七类。

3. **输出必需/可替代/可省清单**
   - 完成标准: 每件器具说明功能，不只列名称。

4. **提示场景边界**
   - 完成标准: 若用户要求简化，转入 `contextual-simplification`。

## B — 边界

- 不做古董鉴价或真伪判断。
- 不把器具清单当成现代泡茶的唯一标准。
- 不鼓励为“复古”牺牲安全和卫生。

### 失败模式

- 把器具当装饰，忽略功能。
- 把场景可省误解成任意省略。

### 盲点 / 时代局限

- 部分器具对应唐代煮茶法，现代冲泡法未必需要。
- 部分材质判断来自古代经验，现代材料安全需另行确认。
