# 品饮场景安排

## R — 来源

源文件: `chajing-skill/source/chajing.md`。

依据《六之饮》中“珍鲜馥烈者，其盌数三；次之者，盌数五”，以及人数至五、至七时的分盏规则；并参考《十之图》将全书要点陈列于座隅的场景意识。

## I — 方法论骨架

陆羽的品饮不是无限供应，而是按茶汤精华、人数和热度控制。
好茶适合少盏，次一等可多盏；人多则要调整盏数和炉数。
分饮要关注沫饽均匀和乘热连饮，因为冷后精华会散。
《十之图》说明茶事还需要现场提示和全局记忆，让参与者看到源、具、造、器、煮、饮、事、出、略的整体。
因此品饮安排不是“倒给大家喝”，而是茶汤状态、人数、器具和教学/展示场景的组合设计。

## A1 — 书中的应用

### 案例: 三盏与五盏
- **问题**: 茶汤该分几盏，如何照顾人数。
- **方法论的使用**: 作者按茶汤品质和人数给出盏数规则。
- **结论**: 分饮应服务于最佳滋味，而不是越多越好。
- **结果**: 形成品饮场景的数量控制。

## A2 — 触发场景

1. 用户要安排一场《茶经》品饮课。
2. 用户问几个人、几盏、如何分饮更合适。
3. 用户要把煮茶结果转成可体验的茶席流程。

### 语言信号

- “几个人怎么分茶”
- “《茶经》三盏五盏”
- “品饮课怎么安排”
- “茶汤冷了还好吗”

### 与相邻 skill 的区分

- 与 `water-fire-brewing-method` 的区别: 水火 skill 负责煮出茶，本 skill 负责分饮和体验。
- 与 `contextual-simplification` 的区别: 本 skill 设计场景，不判断器具是否可省。

## E — 可执行步骤

1. **确认参与人数和目标**
   - 完成标准: 明确人数、教学/品饮/展示目标、是否要复原古法。

2. **决定盏数与分配**
   - 完成标准: 按茶汤品质、人数和炉数给出三盏/五盏/加炉建议。

3. **安排品饮次第**
   - 完成标准: 包含均沫、乘热、补缺人、讲解节点。

4. **输出现场提示**
   - 完成标准: 给出可陈列或讲解的《茶经》全局图谱要点。

## B — 边界

- 不做现代茶会商业营销方案。
- 不把古法盏数当作所有茶会的硬规定。
- 不在食品安全和公共活动组织上替代专业要求。

### 失败模式

- 人多而茶汤过广，滋味黯淡。
- 冷饮后仍按热饮最佳状态评价。
- 忽略现场讲解，导致参与者只看到动作看不到系统。

### 盲点 / 时代局限

- 唐代盏量、器具和饮茶习惯与现代差异较大。
- 古籍对参与者体验的描述偏文人视角。
