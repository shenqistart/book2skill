# 场景化简化取舍

## R — 来源

源文件: `chajing-skill/source/chajing.md`。

依据《九之略》中野寺山园、松间石上、瞰泉临涧、五人以下等条件下可省器具的规则，以及城邑王公之门“二十四器阙一，则茶废”的反向边界。

## I — 方法论骨架

《九之略》不是偷懒清单，而是条件化简化法。
如果环境已经提供了某个功能，就可以省掉对应器具。
如果人数少、茶可精末、场地接近水源，部分收纳、滤水、罗合、都篮可以不用。
但在正式城市茶事或王公之门，完整性本身就是茶事质量的一部分，缺一器可能破坏系统。
所以简化判断的核心问题是：这个器具的功能是否被场景可靠替代，而不是我想不想带。

## A1 — 书中的应用

### 案例: 野外与城邑的不同标准
- **问题**: 为什么同一套茶事有时可以省器具，有时缺一不可。
- **方法论的使用**: 作者按野寺、松间、泉边、人数等条件列出可废项，又以城邑王公之门设反边界。
- **结论**: 简化取决于场景功能替代。
- **结果**: 形成古籍中少见的“情境化 SOP”。

## A2 — 触发场景

1. 用户在户外、课堂、展演中无法准备完整器具。
2. 用户想知道《茶经》哪些步骤可省。
3. 用户要设计轻量版茶事但不想破坏核心体验。

### 语言信号

- “哪些可以省”
- “九之略”
- “没有完整器具怎么办”
- “课堂演示怎么简化”
- “户外煮茶怎么取舍”

### 与相邻 skill 的区分

- 与 `tool-vessel-system` 的区别: 器具系统先说明完整功能，本 skill 判断取舍。
- 与 `water-fire-brewing-method` 的区别: 水火 skill 说明理想流程，本 skill 判断场景替代。

## E — 可执行步骤

1. **识别场景**
   - 完成标准: 明确野外、课堂、展演、城市正式茶席、人数和资源限制。

2. **列出待省项目**
   - 完成标准: 把用户缺少的器具/步骤逐项列出。

3. **按功能替代判断**
   - 完成标准: 每项输出“可省 / 可替代 / 不宜省 / 数据不足”，并说明替代条件。

4. **输出最低可行茶事方案**
   - 完成标准: 保留茶、水、火、煮饮、分饮和清洁安全的关键节点。

## B — 边界

- 不用于随意降低品质或省掉安全步骤。
- 不把野外可省规则套到正式展示场景。
- 不替代现代活动安全与卫生要求。

### 失败模式

- 把“略”理解成删减，而不是功能替代。
- 忽略城邑正式场景对完整性的要求。
- 省掉清洁、火源安全、饮用安全等现代底线。

### 盲点 / 时代局限

- 《九之略》的场景来自唐代茶事环境，现代教室/展馆需要转换。
- 部分器具功能可由现代工具替代，但需说明不是原制复原。
