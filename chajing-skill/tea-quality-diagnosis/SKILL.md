# 茶叶品质与风险判断

## R — 来源

源文件: `chajing-skill/source/chajing.md`。

依据《一之源》中产地、野园、阴阳、芽叶形态和“采不时，造不精，杂以卉莽”的风险判断；《三之造》中成茶形态与精腴/瘠老判断；《八之出》中产地区分。

## I — 方法论骨架

《茶经》的品质判断不是只看外观，也不是只看产地。
它把茶叶质量放在一个链条里：生长环境、采摘时机、芽叶状态、制作是否精细、是否混杂异物、最后才看成茶形态。
好的判断要同时问“它从哪里来”“什么时候采”“采的是什么状态”“怎么做成”“有没有风险”。
产地排名可以作为历史线索，但不能直接替代现代批次判断。
如果用户只给现代商品名，agent 应要求补充茶样描述、工艺、采摘季节和冲泡/品饮反馈。

## A1 — 书中的应用

### 案例 1: 生长环境分层
- **问题**: 何处所产茶更可取。
- **方法论的使用**: 以烂石、砾壤、黄土等环境分层，并比较野者、园者、阳崖阴林等条件。
- **结论**: 品质判断先看自然条件和生态位置。
- **结果**: 形成“产地不是地名，而是生长条件”的判断方式。

### 案例 2: 成茶形态辨精瘠
- **问题**: 成茶外观如何反映制作和原料。
- **方法论的使用**: 用胡人靴、浮云出山、竹箨、霜荷等形态比喻区分精腴与瘠老。
- **结论**: 外观必须回扣采制状态，不可只凭光黑平正判断。
- **结果**: 避免把表面好看误判为真正好茶。

## A2 — 触发场景

1. 用户给出茶样描述，希望判断品质线索。
2. 用户问某茶是否“采得好、做得好、有没有风险”。
3. 用户想用《茶经》解释产地、芽叶、形态与品质的关系。

### 语言信号

- “这款茶按《茶经》怎么看”
- “这个茶样好不好”
- “为什么外观好看不一定好”
- “采不时、造不精是什么意思”

### 与相邻 skill 的区分

- 与 `tea-processing-workflow` 的区别: 本 skill 诊断品质线索；流程缺陷的逐步检查交给制茶流程 skill。
- 与 `water-fire-brewing-method` 的区别: 本 skill 评估茶本身；煮饮过程另行处理。

## E — 可执行步骤

1. **收集茶样信息**
   - 完成标准: 至少获得产地/环境、采摘季节、芽叶状态、制作方式、外观或口感中的两类信息。
   - 判停条件: 信息不足时输出“无法确认”，并列出需要补充的数据。

2. **按《茶经》链条诊断**
   - 完成标准: 分别检查生长环境、采摘时机、制作精度、混杂风险、成茶形态。

3. **输出品质判断等级**
   - 完成标准: 给出“偏优 / 中性待证 / 有风险 / 无法判断”，并逐项说明依据。

4. **标出现代边界**
   - 完成标准: 明确这不是现代检测、价格或医学结论。

## B — 边界

- 不用于出具现代商品等级、价格或真假鉴定。
- 不把唐代产区排名直接套到现代茶叶。
- 不根据单一外观词硬判品质。

### 作者在书中警告的失败模式

- 采摘不合时、制作不精、混杂草木，饮之可能成害。
- 只以光黑平正或皱黄坳垤判断好坏，是低层次鉴别。

### 作者的盲点 / 时代局限

- 唐代制茶和饮茶体系不同于现代六大茶类。
- 古代功效描述不等于现代医学证据。
