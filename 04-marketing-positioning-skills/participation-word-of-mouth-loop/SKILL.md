# 参与感口碑循环

## R — 原文 (Reading)

> “参与感就是台风。”
>
> “产品活动化，活动产品化。”
>
> 来源：《参与感》序言、产品篇。

## I — 方法论骨架 (Interpretation)

口碑不是把用户当传播渠道，而是让用户在产品、品牌、活动和服务中真实参与，并看到自己的参与产生结果。

用户参与越早、反馈越可见、身份越清楚，越可能把产品当成自己的选择来传播。
活动如果只是抽奖，就会在活动结束后消失。
活动产品化，是把参与动作沉淀成可重复机制。
产品活动化，是让产品更新、反馈、发布和服务本身具备传播话题。
真正的口碑循环由体验、参与、反馈、身份和传播共同构成。

这个 skill 的核心是把“做推广”改写成“设计用户愿意参与并愿意讲述的循环”。

## A1 — 书中的应用 (Past Application)

### 案例 1：小米用户参与产品

- **问题**：手机和系统研发复杂，但小米希望让用户参与产品改进。
- **方法论的使用**：通过论坛、反馈、版本迭代和用户荣誉感，让用户意见进入产品。
- **结论**：参与感可以连接研发、营销和服务。
- **结果**：用户不只是消费者，也成为传播和改进的参与者。

### 案例 2：活动产品化

- **问题**：营销活动常常一次性爆发后消散。
- **方法论的使用**：把活动流程、用户反馈和分享动作设计成可复用产品机制。
- **结论**：活动应沉淀资产，而不是只换短期曝光。
- **结果**：口碑变成长期循环。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户要设计社群、用户共创、种子用户计划或粉丝运营。
2. 用户希望提高自然口碑，而不是只买流量。
3. 用户想把活动变成持续机制。
4. 用户担心用户参与只是形式主义。

### 语言信号

- “怎么让用户自发传播？”
- “我们想做社群共创。”
- “这个活动怎样沉淀成长期机制？”
- “怎么让用户有参与感？”

### 与相邻 skill 的区分

- 与 `contagious-trigger-design` 的区别：本 skill 设计参与循环；后者设计内容传播要素。
- 与 `influence-defense-screen` 的区别：本 skill 强调自愿参与，不做操纵式转发。

## E — 可执行步骤 (Execution)

1. **确认真实价值基础**
   - 完成标准：说明用户为什么愿意使用、反馈或推荐，而不是只被奖励驱动。

2. **选择参与节点**
   - 完成标准：在产品、品牌、活动、服务中选出 1-3 个用户可真实影响的节点。

3. **设计反馈闭环**
   - 完成标准：用户能看到意见被采纳、被回应、被展示或被转化为产品变化。

4. **建立身份和荣誉**
   - 完成标准：定义用户参与后的身份、权益、展示和社区位置。

5. **沉淀为可复用机制**
   - 完成标准：把一次性活动改成周期、模板、工具、榜单、发布节奏或产品功能。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 产品体验差，只想靠用户热情掩盖问题。
- 目标是刷好评、诱导转发、伪造社区活跃。
- 用户反馈无法被产品或服务真实响应。

### 作者在书中警告的失败模式

- 只把参与感当营销话术。
- 活动结束后没有沉淀。
- 用户反馈没有进入产品和服务。

### 作者的盲点 / 时代局限

- 小米早期增长、粉丝文化和手机品类具备特殊条件，不能机械复制到低参与、低频或高监管品类。
- 社群运营需要成本和治理，不能只看传播收益。
