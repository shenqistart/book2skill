# 影响力伦理与防操纵筛查

## R — 原文 (Reading)

> “互惠。”
>
> “承诺和一致。”
>
> “稀缺。”
>
> 来源：《影响力》各原则章节。

## I — 方法论骨架 (Interpretation)

影响力原则之所以有效，是因为人类在复杂环境中会使用心理捷径。

这些捷径可以帮助用户快速判断，也可以被营销滥用。
互惠可以是真诚赠予，也可以是强迫亏欠。
承诺一致可以帮助用户坚持目标，也可以诱导逐步套牢。
社会认同可以降低不确定性，也可以伪造成虚假热度。
权威可以提供专业证据，也可以变成伪专家背书。
稀缺可以提醒真实机会成本，也可以制造虚假恐慌。

这个 skill 的核心是把影响力原则反向用作伦理筛查和用户保护。

## A1 — 书中的应用 (Past Application)

### 案例 1：互惠原则

- **问题**：人们收到礼物或让步后，容易产生回报压力。
- **方法论的使用**：识别赠予是否真实自愿，是否被设计成心理债务。
- **结论**：互惠可以建立关系，也可能形成不当压力。
- **结果**：营销方案要区分服务和诱导。

### 案例 2：稀缺原则

- **问题**：人们面对稀缺机会时会提高价值感和紧迫感。
- **方法论的使用**：检查稀缺是否真实、是否清楚说明条件。
- **结论**：虚假倒计时、虚构库存和假限量属于操纵风险。
- **结果**：影响力原则必须接受真实性筛查。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户计划使用限时、限量、专家背书、用户评价、免费礼物、承诺打卡等机制。
2. 用户担心营销方案是否暗黑、操纵或不透明。
3. 用户要审查竞品或页面是否使用不当影响力。
4. 用户希望提高转化但不想伤害信任。

### 语言信号

- “这个倒计时会不会太套路？”
- “能不能用专家背书提高信任？”
- “这些用户评价是否足够透明？”
- “这个转化设计有没有操纵感？”

### 与相邻 skill 的区分

- 与 `behavior-change-lever-stack` 的区别：后者设计行为改变；本 skill 审查影响力原则是否正当。
- 与 `participation-word-of-mouth-loop` 的区别：参与感应建立自愿关系，本 skill 防止伪参与和诱导传播。

## E — 可执行步骤 (Execution)

1. **识别影响力原则**
   - 完成标准：标出方案中使用了哪些原则：互惠、承诺、社会认同、喜好、权威、稀缺。

2. **检查真实性**
   - 完成标准：每个原则都有可验证事实，不能伪造库存、评价、专家或热度。

3. **检查透明和自愿**
   - 完成标准：用户知道自己在接受什么、承诺什么、付出什么，并能自由退出。

4. **评估用户利益**
   - 完成标准：说明该设计是否帮助用户做更好决定，还是只提高商家收益。

5. **输出红黄绿结论**
   - 完成标准：绿色可用，黄色需改写或补证据，红色应移除。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只讨论品牌审美，不涉及行为影响。
- 用户要求绕过平台规则、制造虚假评价或伪造权威。
- 高风险行业需要法律、医学或金融合规判断。

### 作者在书中警告的失败模式

- 自动化心理反应被他人利用。
- 社会认同在不确定时被放大。
- 稀缺和权威让用户降低审查。

### 作者的盲点 / 时代局限

- 数字平台中的算法推荐、数据追踪和个性化诱导比原书案例更复杂。
- 伦理判断需要结合行业规范和用户弱势程度。
