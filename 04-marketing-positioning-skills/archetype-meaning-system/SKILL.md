# 品牌原型意义系统

## R — 原文 (Reading)

> “原型意象：伟大品牌的心跳。”
>
> “发现品牌的原型意义。”
>
> 来源：《如何让品牌直击人心》第2章、第16章。

## I — 方法论骨架 (Interpretation)

品牌意义不是一组随便挑选的形容词，而是品牌在用户心理中扮演的稳定角色。

原型方法把品牌连接到深层欲望，如安全、探索、智慧、力量、归属、爱、愉悦、自由、改变、掌控、创造、关怀等。
选择原型不是为了包装故事，而是为了让产品本质、用户欲望、视觉语言、语气、服务和组织行为保持一致。
如果品牌今天像英雄，明天像爱人，后天像智者，用户会感到人格混乱。
原型只有与品类、证据和文化机会匹配，才会成为长期意义资产。

这个 skill 的核心是把“品牌调性”改写成“品牌在用户欲望系统中扮演什么角色”。

## A1 — 书中的应用 (Past Application)

### 案例 1：12 种原型

- **问题**：品牌需要超越功能利益，建立深层意义。
- **方法论的使用**：用天真者、探索者、智者、英雄、反叛者、魔法师、普通人、爱人、小丑、照顾者、创造者、统治者等原型识别品牌角色。
- **结论**：原型让品牌意义有稳定结构。
- **结果**：品牌故事、符号和体验可以围绕同一角色展开。

### 案例 2：北极星

- **问题**：品牌故事容易碎片化，难以统一组织行为。
- **方法论的使用**：寻找品牌的原型意义作为北极星。
- **结论**：真正的品牌意义必须能指导表达和行动。
- **结果**：原型从文案工具升级为意义管理系统。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户想确定品牌人格、品牌故事或长期调性。
2. 用户发现品牌表达前后不一致。
3. 用户需要让产品、视觉、文案、服务和活动共享同一意义。
4. 用户想判断某个原型是否适合目标用户欲望。

### 语言信号

- “我们的品牌应该是什么人格？”
- “品牌故事怎么讲才不空？”
- “这个调性和用户欲望匹配吗？”
- “我们到底是英雄、探索者还是照顾者？”

### 与相邻 skill 的区分

- 与 `brand-proof-fit-system` 的区别：先检查定位和证据，再用本 skill 统一意义和人格。
- 与 `cultural-era-opportunity-map` 的区别：本 skill 看心理原型；后者看社会文化机会。

## E — 可执行步骤 (Execution)

1. **确认品牌基础**
   - 完成标准：明确品类、目标用户、核心承诺和证据，避免空造人格。

2. **识别用户深层欲望**
   - 完成标准：列出用户在该场景中追求的安全、自由、掌控、归属、成长、愉悦等欲望。

3. **选择候选原型**
   - 完成标准：提出 1 个主原型和最多 1 个辅助原型，并说明为什么排除其他原型。

4. **映射表达系统**
   - 完成标准：把原型落实到故事、语气、视觉、服务、活动和产品细节。

5. **检查一致性和风险**
   - 完成标准：识别原型冲突、文化误读、空洞故事和组织行为不一致。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只要一组品牌形容词或短期 campaign 人设。
- 产品体验和品牌承诺明显不一致。
- 用户想借原型制造虚假身份、煽动偏见或蹭文化符号。

### 作者在书中警告的失败模式

- 原型与产品本质不匹配。
- 品牌故事没有组织行为支撑。
- 把原型当装饰而非意义管理。

### 作者的盲点 / 时代局限

- 原型理论容易泛化，必须结合真实用户研究和文化语境。
- 12 原型不是穷尽分类，也不应压制品牌的复杂性。
