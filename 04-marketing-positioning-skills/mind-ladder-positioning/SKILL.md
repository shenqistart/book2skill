# 心智阶梯定位

## R — 原文 (Reading)

> “定位不是你对产品做什么。”
>
> “竞争的终极战场是顾客心智。”
>
> 来源：《定位》第1章；《升级定位》第4章。

## I — 方法论骨架 (Interpretation)

定位不是把产品优点全部讲出来，而是在顾客已经拥挤的心智里找到一个可进入、可记住、可防守的位置。

先问顾客如何分类，而不是企业如何分类。
再问这个品类里顾客已经记住谁、按什么顺序记住、空位在哪里。
如果没有空位，就不要硬说“更好”，而要考虑对立定位、聚焦子品类或重新定义品类。
定位越清楚，传播越短；定位越含混，广告越需要解释。

这个 skill 的核心是把“我想表达什么”改写成“顾客心智里还有哪个位置能被我占住”。

## A1 — 书中的应用 (Past Application)

### 案例 1：长岛银行

- **问题**：地方银行面对纽约大银行竞争，难以用规模和实力取胜。
- **方法论的使用**：先绘制潜在客户心智地图，找出本地性和亲近性的可占据位置。
- **结论**：不与大银行比“大”，而在顾客心智中强化“长岛自己的银行”。
- **结果**：定位从广告说法变成选择战场。

### 案例 2：个人职业定位

- **问题**：个人在职业市场中也会被简化记忆。
- **方法论的使用**：定义自己希望占据的心智位置，避开无名陷阱和品牌延伸陷阱。
- **结论**：职业定位需要清楚标签、清楚优势和可证明的选择理由。
- **结果**：定位方法从品牌扩展到个人和机构。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户说“我们的品牌定位不清楚”“卖点很多但讲不明白”。
2. 用户想比较多个竞品，判断自己该占据哪个差异位置。
3. 用户准备做品牌升级、改名、slogan 或官网首页，但还没有定位判断。
4. 用户把定位等同于“高级感、年轻化、专业化”等模糊形容词。

### 语言信号

- “我们到底该怎么定位？”
- “用户为什么要记住我们？”
- “这个品牌应该代表什么？”
- “竞品都这么说，我们还能占什么位置？”

### 与相邻 skill 的区分

- 与 `category-entry-design` 的区别：本 skill 先看现有心智阶梯；如果阶梯没有空位，再转新品类入口。
- 与 `brand-proof-fit-system` 的区别：本 skill 选择位置；后者检查这个位置是否有证据和经营配称。

## E — 可执行步骤 (Execution)

1. **定义心智场景**
   - 完成标准：明确目标顾客、购买场景、替代方案和顾客使用的品类语言。

2. **画出心智阶梯**
   - 完成标准：列出顾客最可能先想到的 3-7 个品牌、概念或替代方案。

3. **识别可进入位置**
   - 完成标准：标出第一、第二、对立、细分、区域、场景或人群空位。

4. **压缩定位句**
   - 完成标准：用“对谁，在什么品类中，因为何种关键差异，所以代表什么”写成一句话。

5. **做反向测试**
   - 完成标准：说明这个位置不代表什么，以及哪些竞品不能轻易抢走。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只是要生成广告语、海报标题或短视频脚本。
- 产品价值、目标用户和替代方案都没有基本事实输入。
- 问题核心是伦理、合规、投放预算或平台规则。

### 作者在书中警告的失败模式

- 把定位当成产品说明，而不是顾客心智位置。
- 用品牌延伸稀释原有位置。
- 试图在过度传播环境中解释太多信息。

### 作者的盲点 / 时代局限

- 经典定位来自大众媒体时代，今天的心智可能被搜索、推荐流、社区和私域共同塑造。
- 心智位置必须被产品体验兑现，否则定位会变成空话。
