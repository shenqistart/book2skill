# 流行引爆点诊断

## R — 原文 (Reading)

> “个别人物法则。”
>
> “附着力因素。”
>
> “环境威力法则。”
>
> 来源：《引爆点》。

## I — 方法论骨架 (Interpretation)

流行不是平均地从大众中长出来，而是在关键人、强附着信息和合适环境共同作用下越过临界点。

关键人包括连接员、内行和推销员，他们分别负责跨圈层连接、信息可信度和说服力。
附着力决定信息是否被记住、理解并愿意重复。
环境威力决定小线索和社会场景是否让行为更容易发生。
如果只有内容好，没有关键人；或只有达人，没有附着信息；或环境不支持，流行都很难引爆。

这个 skill 的核心是把“为什么没火”拆成网络、人、信息和环境四个断点。

## A1 — 书中的应用 (Past Application)

### 案例 1：关键少数

- **问题**：某些信息为什么能跨圈层扩散。
- **方法论的使用**：寻找连接员、内行和推销员在传播网络中的不同作用。
- **结论**：少数关键人物可以放大扩散路径。
- **结果**：传播策略不再只看总曝光，而看节点质量。

### 案例 2：环境威力

- **问题**：同样的信息或行为，在不同环境中效果差异巨大。
- **方法论的使用**：检查小环境线索、社会规范和场景压力。
- **结论**：环境能显著改变行为发生概率。
- **结果**：引爆不只是内容和人，还包括场景设计。

## A2 — 触发场景 (Future Trigger)

### 用户会在什么情境下需要这个 skill?

1. 用户想判断一个产品或观点是否可能跨圈层流行。
2. 用户已有传播内容，但不知道该找哪些关键人。
3. 用户发现局部圈层很热，却无法进入大众。
4. 用户要分析某个现象为什么突然爆发。

### 语言信号

- “这个东西有引爆点吗？”
- “为什么只在小圈子火？”
- “应该找什么样的关键人传播？”
- “这个趋势为什么突然起来？”

### 与相邻 skill 的区分

- 与 `contagious-trigger-design` 的区别：后者看内容本身可传播性；本 skill 看网络、关键人和环境临界。
- 与 `cultural-era-opportunity-map` 的区别：本 skill 解释扩散临界；后者判断深层社会文化机会。

## E — 可执行步骤 (Execution)

1. **定义扩散对象和边界**
   - 完成标准：明确要扩散的是产品、观点、行为、活动还是文化符号，以及目标圈层。

2. **识别关键人结构**
   - 完成标准：列出连接员、内行、推销员候选及其证据。

3. **检查附着力**
   - 完成标准：说明信息是否易懂、易记、易复述、与行动相关。

4. **检查环境威力**
   - 完成标准：识别当前场景、规范、平台机制或社会事件是否支持扩散。

5. **判断临界缺口**
   - 完成标准：指出缺的是关键人、附着力、环境，还是三者之间的连接。

## B — 边界 (Boundary)

### 不要在以下情况使用此 skill

- 用户只想优化单条内容标题或短视频脚本。
- 没有传播网络或目标圈层信息。
- 用户想伪造流行、制造水军或操纵舆论。

### 作者在书中警告的失败模式

- 以为大众流行是平均传播。
- 忽略少数关键人的不同角色。
- 低估环境线索对行为的影响。

### 作者的盲点 / 时代局限

- 今天平台算法可能替代或放大关键人角色，需要结合具体平台数据。
- 流行诊断容易事后解释，发布前只能给出假设和验证路径。
