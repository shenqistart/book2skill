# 05 增长和流量 — Skill 分类与差异地图

> 这份文档用于回答：增长和流量类 skills 应该按什么顺序调用？书之间是什么关系？

## 总判断

这 9 本书不是同一件事的重复表达，而是增长问题的不同层级。

- 《回头客战略》问：增长能否靠体验、复购和口碑降低流量成本？
- 《增长黑客》问：产品是否已经具备增长资格，实验系统如何启动？
- 《硅谷增长黑客实战笔记》问：北极星、增长模型和团队节奏怎么落地？
- 《精益数据分析》问：当前阶段唯一该盯的指标是什么？
- 《拉新》问：核心获客渠道如何被找到并打穿？
- 《关系飞轮》问：私域和超级用户如何形成关系复利？
- 《小群效应》问：小群、连接者和社群密度如何放大转化和留存？
- 《直播电商的逻辑》问：直播场景下人货场和平台生态是否匹配？
- 《流媒体时代》问：平台如何重写分发权力，内容方如何避免被平台锁死？

## 一级分类

| 分类 | 解决的问题 | 对应 skill | 典型输出 |
|---|---|---|---|
| 复购与口碑 | 流量成本高、客户不复购、口碑弱 | `repeat-customer-experience-loop` | 体验缺口、复购动作、口碑资产 |
| 增长门禁 | 是否应该开始增长攻势 | `growth-readiness-aha-gate` | 啊哈时刻、不可或缺性、留存门槛 |
| 增长系统 | 增长动作零散、实验缺闭环 | `growth-experiment-operating-system` | 实验节奏、团队角色、学习闭环 |
| 指标与模型 | 指标太多、优先级冲突 | `north-star-growth-model`, `omtm-lean-analytics-focus` | 北极星、增长模型、OMTM |
| 渠道拉新 | 不知道选哪个获客渠道 | `bullseye-channel-selection` | 渠道候选、低成本测试、核心渠道 |
| 关系复利 | 私域、超级用户、社群转化 | `relationship-flywheel-private-domain`, `small-group-community-leverage` | 关系类型、小群结构、连接者 |
| 平台环境 | 直播、内容、媒体和平台分发 | `live-commerce-fit-map`, `platform-power-shift-strategy` | 平台适配、分发权力、依赖边界 |

## 推荐调用顺序

1. 先用 `repeat-customer-experience-loop` 判断增长是否能沉淀为复购。
2. 再用 `growth-readiness-aha-gate` 判断是否具备放大资格。
3. 用 `north-star-growth-model` 和 `omtm-lean-analytics-focus` 定义指标焦点。
4. 用 `growth-experiment-operating-system` 组织实验。
5. 用 `bullseye-channel-selection` 找核心渠道。
6. 如果需要更高复购和转介绍，再用 `relationship-flywheel-private-domain` 和 `small-group-community-leverage`。
7. 如果增长依赖直播、短视频、内容平台或流媒体，再用 `live-commerce-fit-map` 和 `platform-power-shift-strategy` 做平台环境判断。

## 主要张力与调和方式

| 张力 | 表面冲突 | 调和方式 |
|---|---|---|
| 回头客 vs 拉新 | 一个强调复购，一个强调新用户 | 先确认复购和留存，再用拉新放大；否则拉新会变漏水桶 |
| 北极星 vs OMTM | 一个全局统一，一个阶段聚焦 | 北极星定长期方向，OMTM 定当前阶段最该解决的问题 |
| 增长实验 vs 私域关系 | 一个强调快速测试，一个强调长期信任 | 实验不能透支信任；私域实验先定义用户获得的价值 |
| 小群效应 vs 规模化 | 小群重密度，增长要规模 | 用连接者和小群网络扩散，而不是把所有人塞进大群 |
| 直播电商 vs 平台权力 | 直播能转化，平台也会抽取权力 | 做直播前同时检查自有用户资产、供应链和平台依赖 |

## 最短决策树

```text
流量成本高或复购低？
  是 -> repeat-customer-experience-loop
  否 -> 继续

产品是否已经有啊哈时刻和稳定留存？
  否 -> growth-readiness-aha-gate
  是 -> 继续

团队指标混乱？
  是 -> north-star-growth-model -> omtm-lean-analytics-focus
  否 -> 继续

增长动作没有实验闭环？
  是 -> growth-experiment-operating-system
  否 -> 继续

不知道选哪个获客渠道？
  是 -> bullseye-channel-selection
  否 -> 继续

问题涉及私域、超级用户、社群或转介绍？
  是 -> relationship-flywheel-private-domain -> small-group-community-leverage
  否 -> 继续

问题涉及直播电商、内容平台或流媒体分发？
  是 -> live-commerce-fit-map -> platform-power-shift-strategy
```
